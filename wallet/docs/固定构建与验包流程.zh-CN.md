# 固定构建与验包流程

这份文档的目标只有一个：让你以后重编 `wallet/` 时尽量少漂移。

## 1. 先固定 4 件事

以后想稳定重编，至少要固定：

1. 源码提交
2. `JDK / SDK / Build Tools`
3. 构建命令
4. 同一套发布 keystore

## 2. 当前不要再硬记某个旧提交号

这份文档不再硬写“稳定提交 = 某个旧 hash”，因为那种写法很快会过时。

以后如果你要追溯仓库里已经保留的那份 APK，先看：

- `dist/satochip-wallet-release.build-info.txt`

里面的：

- `repo_head`
- `artifact_sha256`
- `version_name`
- `build_time_utc`

才是当前产物真正对应的基准。

如果它和当前 `git rev-parse HEAD` 不同，就说明你手里的 APK 不是当前源码刚重编出来的。

## 3. 当前固定环境

建议固定为：

- `JDK 17`
- `Android SDK 34`
- `Android Build Tools 35.0.0`
- Linux / Ubuntu

环境文档：

- [../../docs/安卓构建环境准备.zh-CN.md](../../docs/安卓构建环境准备.zh-CN.md)

## 4. 当前 release 签名规则

`wallet/` 现在更严格：

- `release` 必须有 `wallet/keystore.properties`
- 没有就直接失败
- 不允许再回退到 `debug` 签名冒充 release

所以你必须长期保存：

- `wallet/keystore.properties`
- 它引用的正式 keystore 文件

最好再额外保留一份离线文字记录，至少记住：

- `storePassword`
- `keyAlias`
- `keyPassword`

如果以后只剩源码、没有这把 keystore：

- 你仍然可以继续编测试 APK
- 但不能继续发“和旧正式钱包同一签名”的 release APK
- GitHub 上的钱包自动正式验包也会因为缺少 secrets 无法继续跑

## 5. 当前推荐构建命令

如果目标是“把当前源码改动编成测试 APK”，先跑：

```bash
./scripts/build_local_ascii.sh debug
```

它会产出：

- `dist/satochip-wallet-debug.apk`
- `dist/satochip-wallet-debug.apk.sha256`

如果目标是“构建当前 release APK”，先配置 `keystore.properties`，再跑：

```bash
./scripts/build_local_ascii.sh release
```

它会在纯英文临时目录里构建，并输出 `dist/satochip-wallet-release.apk`。

### `wallet/` 目录纯英文临时路径构建

```bash
cd wallet
./scripts/build_local_ascii.sh release
```

## 6. 构建后的标准检查

测试当前源码时，至少确认这 2 个文件存在：

- `dist/satochip-wallet-debug.apk`
- `dist/satochip-wallet-debug.apk.sha256`

如果是 release 构建，再确认这 2 个文件存在：

- `dist/satochip-wallet-release.apk`
- `dist/satochip-wallet-release.apk.sha256`

## 7. 验包步骤

### 校验哈希

```bash
sha256sum dist/satochip-wallet-release.apk
cat dist/satochip-wallet-release.apk.sha256
```

两者必须一致。

### 校验来源

记录当前源码状态，避免以后忘记 APK 来自哪份代码：

```bash
git rev-parse HEAD
git status --short
```

如果工作区不是干净的，就把 `git status --short` 的输出和 APK SHA256 一起留档。

## 8. 实机回归清单

不要只看 Gradle 成功。

每次至少验证：

1. 能安装新包
2. 能通过强生物识别
3. 首页 EVM 地址管理正常
4. `Arbitrum` 资产刷新正常
5. EVM 待签名二维码生成正常
6. `BTC` 观察账户导入和同步正常
7. `BTC PSBT` 请求生成正常
8. 树莓派回签扫码正常
9. `WalletConnect v2` 连接正常
10. 锁屏后会话按高安全策略清理

## 9. 最值得记住的一句

以后别靠记忆重编。

每次都按：

- 固定源码
- 固定环境
- 固定命令
- 固定 keystore
- 固定验包步骤
- 正式发布优先用干净工作区和固定签名重新跑 `./scripts/build_local_ascii.sh release`

这样结果才最稳。
