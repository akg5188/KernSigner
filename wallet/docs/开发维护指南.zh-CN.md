# Satochip 钱包开发维护指南

这份文档对应 `wallet/` 目录当前源码。

## 1. 当前项目定位

当前工程不是热钱包，而是：

- `Arbitrum One` 高安全观察钱包
- 树莓派离线签名协调器
- `WalletConnect v2` 高安全协调器
- `BTC` 观察账户原型

手机里不保存私钥。

## 2. 当前明确支持什么

- `Arbitrum One`
- `ETH / USDC / USDT`
- `WalletConnect v2`
- `personal_sign`
- `eth_signTypedData`
- `eth_signTypedData_v3`
- `eth_signTypedData_v4`
- `eth_signTransaction`
- `BTC xpub / ypub / zpub` 导入
- `BTC PSBT` 冷签请求准备与广播

## 3. 当前明确不做什么

- 热钱包私钥
- 内置 DApp 浏览器
- `Hyperliquid` 内置网页
- `wallet_switchEthereumChain`
- `wallet_addEthereumChain`
- 锁屏后继续保留活跃 DApp 会话

## 4. 当前安全模型

维护时不要悄悄放宽这些边界：

- 强生物识别是硬门槛
- `ADB / USB 调试`、开发者选项、`Root / Hook`、`test-keys` 都会拦截
- 关键联网主机有白名单和证书 pinning
- `WalletConnect` 只接受已验证、`HTTPS`、主机合法的 DApp
- 可信范围按 `主机 + 链 + 地址` 绑定，且 `24` 小时过期
- 锁屏或退后台后主动清理敏感状态
- `release` 必须使用正式 keystore

## 5. 当前关键文件

- `wallet/app/src/main/java/io/arbitrum/wallet/MainActivity.kt`
  主界面、首页/活动/DApp、请求展示、广播确认
- `wallet/app/src/main/java/io/arbitrum/wallet/MainViewModel.kt`
  观察地址、BTC 账户、签名请求、广播、DApp 状态
- `wallet/app/src/main/java/io/arbitrum/wallet/BiometricGateActivity.kt`
  设备环境检查与生物验证
- `wallet/app/src/main/java/io/arbitrum/wallet/WalletConnectBridge.kt`
  `WalletConnect v2` 接桥
- `wallet/app/src/main/java/io/arbitrum/wallet/WalletConnectRequestCodec.kt`
  DApp 请求校验与转码
- `wallet/app/src/main/java/io/arbitrum/wallet/BitcoinTransferService.kt`
  BTC 数据同步、转账准备、广播
- `wallet/app/src/main/java/io/arbitrum/wallet/TrustedNetwork.kt`
  白名单与证书 pinning

## 6. 当前构建规则

当前源码测试包：

```bash
./scripts/build_local_ascii.sh debug
```

当前 release 包：

```bash
./scripts/build_local_ascii.sh release
```

如果只想确认 Gradle 本身能跑，可以在 `wallet/` 下直接用：

```bash
./gradlew :app:assembleDebug
```

当前 `release` 构建不会再回退到 debug 签名。

## 7. 每次改完至少回归这些

1. 能通过生物验证进入
2. 首页 EVM 地址管理正常
3. `Arbitrum` 资产刷新正常
4. 能生成一笔 EVM 待签名请求
5. 能导入 `BTC xpub / ypub / zpub`
6. 能同步 BTC 账户
7. 能准备一笔 `BTC PSBT`
8. 能扫回树莓派签名结果
9. `WalletConnect v2` 能建立连接
10. 锁屏后敏感状态会按策略清理

## 8. 最容易写错的地方

- 不要再按“内置 `Hyperliquid` 浏览器”思路维护
- 不要把 `wallet` 和 `app` 的签名规则写反
- 不要把当前链支持夸成“全 EVM 多链钱包”，源码里现在只有 `Arbitrum One`
