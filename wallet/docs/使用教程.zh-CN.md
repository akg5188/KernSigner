## 1. 先理解这不是热钱包

当前钱包不会把私钥存到手机里。

它负责：

- 保存观察地址
- 准备树莓派签名请求
- 扫描树莓派回签结果
- 在手机上人工确认广播
- 处理高安全 `WalletConnect`
- 保留 Web3 兼容桥接 / 高级导入工具

## 2. 启动前的安全门槛

当前高安全模式下，下面任一条件不满足都可能被拦住：

- 已配置强生物识别
- 没开 `USB 调试`
- 没开开发者选项
- 没有 `Root / Magisk / Hook`
- 系统不是 `test-keys`

被拦住时，界面会直接告诉你原因。

## 3. 顶部 3 个标签页

当前就是这 3 个：

- `首页`
- `活动`
- `DApp`

## 4. 首页怎么用

首页当前分两块模式：

- `Arbitrum`
- `BTC`

### 4.1 `Arbitrum` 模式

可以做：

- 填 `派生路径`
- 连接树莓派
- 手动添加观察地址
- 选择当前地址
- 查看资产与最近活动
- 发起 EVM 转账待签名请求

### 4.2 `BTC` 模式

可以做：

- 扫码导入 `xpub / ypub / zpub`
- 粘贴导入 `xpub / ypub / zpub`
- 查看已导入账户
- 同步余额、活动、下一收款地址
- 准备 `BTC PSBT` 冷签请求

当前主网 `BTC` 观察账户同步会优先直连 `Electrum`。
正常情况下，余额和最近活动会比旧的逐地址 HTTP 扫描快很多；如果 `Electrum` 临时不可用，才会自动回退到旧同步路径。

## 5. 树莓派配合流程

### EVM

1. 在手机里准备转账或 DApp 请求
2. 手机显示待签名二维码
3. 树莓派扫描并签名
4. 手机扫回结果
5. 人工确认后广播

### BTC

1. 先从树莓派导出 `xpub / ypub / zpub`
2. 手机导入观察账户
3. 同步链上状态
4. 准备 `PSBT`
5. 树莓派扫描并签名
6. 手机扫回 `tx` 结果并广播

## 6. `DApp` 页怎么用

当前 `DApp` 页不是浏览器，但保留两类入口：独立钱包用的 `WalletConnect / DApp 签名`，以及 OKX / Bitget / TP 高密度二维码中转。

支持：

- 扫码导入 `WalletConnect v2`，用于独立钱包连接 DApp
- 扫描 TP / OKX / Bitget 的高密度钱包二维码
- 自动转成树莓派更容易扫的低密度中转二维码
- 树莓派签完后，让原钱包直接扫树莓派结果码

### 6.1 `OKX Wallet / Bitget Wallet`

连接钱包：

1. 树莓派首页点 `连接钱包 -> Web3钱包`
2. 选择 `OKX钱包` 或 `Bitget钱包`
3. 选择 `智能卡账户` 或 `已加载助记词`
4. 树莓派按钱包类型直接显示 Keystone 兼容连接二维码
   - `OKX`：动态 `crypto-multi-accounts`
   - `Bitget`：单张静态 `crypto-multi-accounts`
5. `OKX Wallet / Bitget Wallet` 选择添加 `Keystone` 硬件钱包并扫描

签名中转：

1. `OKX / Bitget / TP` 显示高密度签名请求二维码
2. 安卓观察钱包进入 `DApp` 页
3. 点 `扫描二维码`，必要时也可以改用 `相册导入` 或 `导入视频`
   - 如果签名请求二维码就在本机屏幕上，可以先录屏，再把录屏视频直接导入识别
4. 手机自动显示低密度中转二维码
5. 树莓派首页点 `扫码签名` 并扫描手机
6. 树莓派核对摘要并签名
7. Web3 钱包直接扫树莓派结果码

说明：

- 安卓观察钱包不再负责生成 Web3 连接码
- 安卓观察钱包这里只负责高密度签名请求中转，不负责保存 Keystone 公开账户

当前策略：

- 只接受已验证、`HTTPS`、主机合法的 DApp
- 可信范围绑定到 `主机 + 链 + 地址`
- 超过 `24` 小时失效
- 锁屏或退后台会主动清理会话

## 7. 当前 BTC 支持范围

按当前源码，手机端已经有这些真实能力：

- 导入 `BTC xpub / ypub / zpub`
- 同步 `BTC` 余额与活动
- 生成待树莓派签名的 `PSBT`
- 扫回树莓派返回的已签名交易并广播

当前体验上的变化：

- 主网同步优先走 `Electrum`
- 首次导入时不再先傻等大批量地址探测才显示余额
- `BlueWallet` 路线里，树莓派回签二维码已经改回兼容连续扫描的动画输出

但它还不是完整多链钱包，也不是闪电钱包。

## 8. 常见问题

### 8.1 为什么没有内置 `Hyperliquid`

因为当前代码已经移除了内置浏览器路线。

### 8.2 为什么不自动从系统剪贴板读敏感内容

这是高安全设计的一部分，目的是减少剪贴板污染和恶意 App 读取风险。

### 8.3 为什么锁屏后 DApp 会断

这是当前高安全策略的正常行为，不是故障。

### 8.4 为什么广播前还要我再确认一次

因为当前设计就是：

- 树莓派离线签名
- 手机二次人工确认
- 再广播

不会为了省一步而自动替你发链上交易。
