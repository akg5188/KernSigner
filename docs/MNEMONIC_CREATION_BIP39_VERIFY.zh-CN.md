# 助记词创建与 BIP39 网站验证指南

这份文档给第一次使用的人看。目标是：你按设备上的创建方法生成助记词后，可以用 BIP39 网站验证“同一份熵会得到同一组助记词”。

重要提醒：真实钱包助记词不要输入联网网页。真钱包验证请使用离线版 BIP39 工具，或者只用测试助记词练习。

## 先记住三句话

- 助记词不是随便排列的单词。BIP39 会先有一段“原始熵”，再自动加校验位，最后变成 12/15/18/21/24 个英文单词。
- 只要“原始熵”和“英文词库”一样，KernSigner 和 BIP39 网站生成出来的助记词就应该完全一样。
- 最稳的验证方法是：在 KernSigner 上查看“原始熵”的十六进制，把它填到 BIP39 网站的 Entropy 里，Mnemonic Length 选择 `Use Raw Entropy`。

## 创建入口

常见入口：

1. 进入首页。
2. 点 `助记词`。
3. 点 `创建`。
4. 选择创建方式，例如 `骰子`、`抛硬币`、`手动`、`相机`、`16进制`、`D20`、`扑克牌`、`智能卡`。
5. 按页面提示选择词数并输入随机材料。
6. 生成后认真抄写助记词，并完成设备上的确认流程。

建议新手优先用 `12 词` 练习。未来真正进入生产使用前，可以选择 `24 词` 提高熵强度；当前测试资金验收版只用测试助记词、测试网资产或你明确可承受损失的小额验收资金。

## 词数和输入长度

| 词数 | 原始熵 | 十六进制长度 | 抛硬币长度 |
| --- | ---: | ---: | ---: |
| 12 词 | 128 位 | 32 个十六进制字符 | 128 个 0/1 |
| 15 词 | 160 位 | 40 个十六进制字符 | 160 个 0/1 |
| 18 词 | 192 位 | 48 个十六进制字符 | 192 个 0/1 |
| 21 词 | 224 位 | 56 个十六进制字符 | 224 个 0/1 |
| 24 词 | 256 位 | 64 个十六进制字符 | 256 个 0/1 |

十六进制字符只允许 `0-9` 和 `a-f`。大小写都可以。

## BIP39 网站离线准备

推荐工具是 Ian Coleman 的 BIP39 工具。

1. 打开项目地址：`https://github.com/iancoleman/bip39`
2. 下载发布页里的 `bip39-standalone.html`。
3. 断网。
4. 双击打开 `bip39-standalone.html`。
5. 页面里选择 `English`。
6. 勾选 `Show entropy details`，用于显示 Entropy 区域。

只验证测试助记词时，也可以打开在线版：`https://iancoleman.io/bip39/`。真实助记词不要这样做。

## 通用验证方法：用原始熵验证

这个方法适用于所有创建方式，包括相机、D20、智能卡、手动输入。

### 第一步：在 KernSigner 上查看原始熵

创建并加载助记词后：

1. 回到首页。
2. 点 `助记词`。
3. 进入当前已加载的助记词菜单。
4. 点 `原始熵`。
5. 记录页面底部的连续十六进制字符串。

页面可能同时显示：

- `原始输入`：你输入的骰子、硬币、扑克牌等材料。
- `十六进制`：最终用于生成助记词的 BIP39 原始熵。

验证时优先使用 `十六进制` 那一行。

### 第二步：在 BIP39 网站上填入原始熵

1. 打开离线版 BIP39 网站。
2. 勾选 `Show entropy details`。
3. 找到 `Entropy` 输入框。
4. 把 KernSigner 的十六进制原始熵粘进去，空格可以去掉。
5. `Entropy Type` 确认是 `hexadecimal`。
6. `Mnemonic Length` 选择 `Use Raw Entropy (3 words per 32 bits)`。
7. 查看 `BIP39 Mnemonic`。
8. 网站显示的助记词应当和 KernSigner 上的一模一样。

注意：验证 KernSigner 的原始熵时，不要在 BIP39 网站里选择 `12 Words`、`15 Words`、`24 Words` 这类选项。那些选项会让网站先对输入做 SHA256 再截取，结果会不一样。

## 方法一：16进制创建

适合想和 BIP39 网站最直接对照的人。

### KernSigner 上怎么做

1. 进入 `助记词` -> `创建` -> `16进制`。
2. 选择词数。
3. 输入对应长度的十六进制：
   - 12 词输入 32 个字符。
   - 24 词输入 64 个字符。
4. 点 `生成`。
5. 核对并确认助记词。

### BIP39 网站怎么验证

1. 勾选 `Show entropy details`。
2. 在 `Entropy` 输入同一串十六进制。
3. `Entropy Type` 选择 `hexadecimal`。
4. `Mnemonic Length` 选择 `Use Raw Entropy`。
5. 网站的 `BIP39 Mnemonic` 必须和 KernSigner 一样。

### 入门测试

用 12 词测试：

```text
00000000000000000000000000000000
```

预期助记词：

```text
abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon about
```

这只是测试向量，不要当真钱包使用。

## 方法二：抛硬币

适合完全用实物随机性创建。

### KernSigner 上怎么做

1. 进入 `助记词` -> `创建` -> `抛硬币`。
2. 选择词数。
3. 约定正面为 `1`，反面为 `0`，或反过来也行，但要从头到尾一致。
4. 每抛一次就在设备上输入一个 `0` 或 `1`。
5. 输入到页面提示达到目标位数后，点 `完成` 或 `生成`。

### BIP39 网站怎么验证

1. 在 `Entropy` 输入同一串 `0/1`。
2. `Entropy Type` 选择 `binary`。
3. `Mnemonic Length` 选择 `Use Raw Entropy`。
4. 对比 `BIP39 Mnemonic`。

常见错误：多输一个 0 或 1，后面的助记词会完全变掉。

## 方法三：六面骰

适合用普通骰子创建。

### KernSigner 上怎么做

1. 进入 `助记词` -> `创建` -> `骰子`。
2. 选择词数。
3. 投掷六面骰，按顺序输入 `1-6`。
4. 页面会显示熵是否已经足够。
5. 足够后点 `完成` 或 `生成`。

KernSigner 使用和 Ian Coleman BIP39 工具相同的无偏取位规则：

| 骰子点数 | 转成的位 |
| --- | --- |
| 1 | `01` |
| 2 | `10` |
| 3 | `11` |
| 4 | `0` |
| 5 | `1` |
| 6 | `00` |

### BIP39 网站怎么验证

最稳方法：用 KernSigner 的 `原始熵` 十六进制验证。

也可以直接用骰子序列验证：

1. 在 `Entropy` 输入同一串骰子点数，例如 `123456123456...`。
2. `Entropy Type` 选择 `dice`。
3. `Mnemonic Length` 选择 `Use Raw Entropy`。
4. 对比网站生成的助记词。

如果直接用骰子序列不一致，优先检查是否多输入了骰子。KernSigner 会按所选词数取最终需要的那段熵；对新手来说，用 `原始熵` 十六进制验证更不容易错。

## 方法四：二十面骰 D20

适合有 D20 骰子的用户。

### KernSigner 上怎么做

1. 进入 `助记词` -> `创建` -> `D20` 或 `二十面骰`。
2. 选择词数。
3. 每次投掷输入 `1-20` 之间的数字。
4. 数字之间用空格分开，例如：

```text
1 20 7 13 4 9 18 2
```

5. 输入到页面提示足够后，点 `生成`。

### BIP39 网站怎么验证

BIP39 网站没有原生 D20 类型，所以不要直接把 D20 数字粘到 Entropy 里。

正确做法：

1. 在 KernSigner 上打开 `原始熵`。
2. 复制十六进制原始熵。
3. 在 BIP39 网站的 `Entropy` 输入这串十六进制。
4. `Entropy Type` 选择 `hexadecimal`。
5. `Mnemonic Length` 选择 `Use Raw Entropy`。
6. 对比助记词。

## 方法五：扑克牌

适合用洗牌后的实体扑克牌创建。

### KernSigner 上怎么做

1. 进入 `助记词` -> `创建` -> `扑克牌`。
2. 选择词数。
3. 洗牌后从牌堆顶按顺序输入。
4. 牌面格式是“点数 + 花色”：
   - `A` 表示 Ace。
   - `T` 表示 10。
   - `J/Q/K` 表示 J/Q/K。
   - `C` 梅花，`D` 方块，`H` 红桃，`S` 黑桃。
5. 示例：

```text
AH QS 9D TC 2C
```

### BIP39 网站怎么验证

最稳方法：用 KernSigner 的 `原始熵` 十六进制验证。

也可以直接用扑克牌序列验证：

1. 在 `Entropy` 输入同一串牌，例如 `AH QS 9D TC`。
2. `Entropy Type` 选择 `card`。
3. `Mnemonic Length` 选择 `Use Raw Entropy`。
4. 对比助记词。

注意：`10` 要写成 `T`，不能写成 `10H`。例如红桃 10 写 `TH`。

## 方法六：相机

适合不想手动输入很多随机材料时使用。

### KernSigner 上怎么做

1. 进入 `助记词` -> `创建` -> `相机`。
2. 选择词数。
3. 对着有丰富细节的场景，例如桌面、纸张纹理、不同颜色物体。
4. 点屏幕拍摄。
5. 如果提示随机性不足，换一个更复杂的画面重新拍。
6. 设备会显示快照 SHA256，然后点 `继续`。

### BIP39 网站怎么验证

相机方式本质是：

1. 拍照得到画面。
2. 对快照计算 SHA256。
3. 按词数取需要长度的熵。
4. 用 BIP39 生成助记词。

验证时推荐直接用 KernSigner 的 `原始熵` 十六进制：

1. 创建完成后进入 `原始熵`。
2. 复制十六进制。
3. BIP39 网站选择 `hexadecimal` + `Use Raw Entropy`。
4. 对比助记词。

如果你只看到快照 SHA256：

- 12 词：取 SHA256 前 32 个十六进制字符。
- 15 词：取前 40 个十六进制字符。
- 18 词：取前 48 个十六进制字符。
- 21 词：取前 56 个十六进制字符。
- 24 词：使用完整 64 个十六进制字符。

然后按 `16进制创建` 的验证方法填到 BIP39 网站。

## 方法七：手动选择单词

适合已经知道 BIP39 单词，或者要恢复/检查某组词的人。

### KernSigner 上怎么做

1. 进入 `助记词` -> `创建` -> `手动`。
2. 按顺序输入或选择单词。
3. 最后一个词包含校验位，不能随便填。
4. 如果设备提示无效，通常是单词拼错、顺序错、或最后一个词校验不对。

### BIP39 网站怎么验证

1. 在 BIP39 网站找到 `BIP39 Mnemonic`。
2. 粘贴完整助记词。
3. 语言选择 `English`。
4. 页面没有 invalid 警告，说明 BIP39 校验通过。
5. 也可以勾选 `Show entropy details` 查看对应原始熵。

手动方式不是“从随机输入自动创建”，所以验证重点是助记词本身是否符合 BIP39 校验。

## 方法八：智能卡

适合使用 SeedKeeper/Satochip 等智能卡随机数能力。

### KernSigner 上怎么做

1. 接好智能卡和读卡器。
2. 进入 `助记词` -> `创建` -> `智能卡`。
3. 按页面提示输入智能卡 PIN。
4. 选择词数或使用默认词数。
5. 设备从智能卡生成随机秘密，再转换为 BIP39 助记词。

### BIP39 网站怎么验证

1. 创建并加载后，进入 KernSigner 的 `原始熵`。
2. 复制十六进制。
3. BIP39 网站选择 `hexadecimal` + `Use Raw Entropy`。
4. 对比助记词。

如果页面只显示助记词，没有显示智能卡随机数，也可以把助记词粘到 `BIP39 Mnemonic` 字段检查校验。

## BIP39 网站常见设置

验证 KernSigner 生成结果时，一般用这些设置：

| 项目 | 应该选择 |
| --- | --- |
| Mnemonic Language | `English` |
| Show entropy details | 勾选 |
| Entropy Type | 按输入选择：`hexadecimal` / `binary` / `dice` / `card` |
| Mnemonic Length | `Use Raw Entropy` |
| BIP39 Passphrase | 验证助记词本身时留空 |

如果只是检查助记词有效性：

1. 把助记词粘到 `BIP39 Mnemonic`。
2. 不需要填 Entropy。
3. 页面不报 invalid，说明校验通过。

如果要对比地址，还要确保网络和路径一致。例如 BTC 原生隔离见证常见路径是 `m/84'/0'/0'`，测试网是 `m/84'/1'/0'`。路径不同，地址一定不同。

## 不一致时怎么排查

按这个顺序查：

1. 是否用了离线版或同一个 BIP39 工具页面。
2. `Mnemonic Language` 是否是 `English`。
3. `Mnemonic Length` 是否选了 `Use Raw Entropy`。
4. 十六进制是否少了字符或多了空格以外的符号。
5. 抛硬币是否把正反面规则中途反了。
6. 骰子是否多输入或漏输入。
7. 扑克牌的 10 是否写成了 `T`。
8. D20 是否直接粘进网站了。D20 要用 KernSigner 的十六进制原始熵验证。
9. 相机方式是否拿错了 SHA256：12 词只用前 32 个十六进制字符，24 词才用完整 64 个。
10. 是否把密码短语 passphrase 也算进去了。验证助记词本身时 passphrase 留空。

## 最推荐的新手流程

第一次练习：

1. 用 `16进制创建`。
2. 输入全 0 测试熵：

```text
00000000000000000000000000000000
```

3. 看到测试助记词：

```text
abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon about
```

4. 用 BIP39 网站按 `hexadecimal` + `Use Raw Entropy` 验证。
5. 练熟后，再用骰子、硬币、相机或智能卡创建新的测试助记词做恢复演练。
6. 未来如果创建真实助记词，不要联网验证，不要拍照，不要发给任何人；当前固件未过生产门禁前不要导入真实助记词。
