# Kern 连接钱包与扫码签名教程

日期：2026-05-21

这份教程给第一次使用 Kern 的人看。目标是：你拿着开发板、手机钱包或电脑钱包时，知道应该点哪里、扫哪个二维码、输入哪个 PIN、核对哪些信息。

当前 Kern 仍是测试资金验收版，不是已经审计的商业生产钱包。所有 Web3、BTC、智能卡流程都先用小额测试资金验收，不要直接放真实大额资产。

## 先看这 1 分钟

| 事情 | 正确做法 |
|---|---|
| Web3 钱包连接 | 在 OKX、Bitget、MetaMask、Rabby、TokenPocket 里选择 `硬件钱包`、`Connect Hardware Wallet`、`Keystone` 或 `QR-based`，然后扫描 Kern 显示的 Keystone 兼容二维码 |
| 不要做什么 | 不要把 Kern 的助记词、智能卡助记词、私钥导入手机钱包或浏览器插件 |
| BTC 钱包连接 | 把 Kern 显示的 `zpub` 或 `xpub` 导入成只读观察钱包 |
| Web3 签名 | 手机/插件显示待签名二维码，Kern 进入 `扫码签名` 扫码，核对完整信息后签名，再让手机/插件扫回签名结果 |
| BTC 签名 | BTC 钱包生成 PSBT 二维码，Kern 进入 `扫码签名` 扫码，核对收款地址、找零和矿工费后签名，再把签名 PSBT 扫回钱包广播 |
| PIN | 屏幕写 `输入 PIN 码` 通常是开发板 PIN；屏幕写 `智能卡 PIN` 才输入 Satochip/SeedKeeper 卡 PIN |

## Kern 上的签名来源怎么选

Kern 现在有两类签名来源。

| 来源 | 适合什么 | 需要哪个 PIN |
|---|---|---|
| `助记词` | 已经加载到 Kern 开发板内存里的本机助记词 | 开发板 PIN |
| `智能卡` | 插在 USB 读卡器里的 Satochip/SeedKeeper 卡 | 智能卡 PIN |

最简单判断：

| 你要做的事 | 在 Kern 上选什么 |
|---|---|
| 用开发板里已经加载的助记词连接 Web3 | `连接钱包 -> 钱包名 -> 助记词` |
| 用 Satochip 智能卡连接 Web3 | `连接钱包 -> 钱包名 -> 智能卡` |
| 用开发板助记词导出 BTC 观察钱包 | `连接钱包 -> BTC -> 助记词 -> zpub` |
| 用 Satochip 智能卡导出 BTC 观察钱包 | `连接钱包 -> BTC -> 智能卡 -> zpub` |
| Web3 签名时想用开发板助记词 | `扫码签名` 后选 `助记词` |
| Web3 签名时想用 Satochip 卡 | `扫码签名` 后选 `智能卡` |
| BTC PSBT 签名 | 当前验收主线优先用开发板助记词；Satochip BTC 卡片 PSBT 签名仍不作为生产可用功能宣称 |

如果你的固件版本菜单里第一层还有 `Web3钱包` 或 `比特币钱包`，就先进入这一层，再按下面的钱包名继续。当前菜单通常是：

```text
首页 -> 连接钱包 -> OKX / Bitget / imToken / MetaMask / Rabby / TokenPocket / Keystone / BTC
```

## Web3 钱包连接总规则

Web3 商业钱包包括：

```text
OKX Wallet / Bitget Wallet / imToken / MetaMask / Rabby / TokenPocket
```

在这些钱包里连接 Kern 时，一律不要选择：

```text
导入助记词
导入私钥
Import seed phrase
Import private key
创建新钱包后把助记词复制过去
```

必须选择类似下面的入口：

```text
硬件钱包
连接硬件钱包
Connect Hardware Wallet
Keystone
QR-based
冷钱包
扫码连接 Keystone
```

原因很简单：手机钱包和浏览器插件只负责联网、看余额、发起交易和广播交易。私钥仍然留在 Kern 开发板或智能卡里，签名通过二维码来回传递。

## Web3：在 Kern 上生成连接二维码

以 OKX 为例，其他钱包只把钱包名换成 Bitget、imToken、MetaMask、Rabby 或 TokenPocket。

### 用开发板助记词连接

1. 确认已经在 Kern 里加载或创建过助记词。
2. 在 Kern 首页进入：

```text
连接钱包 -> OKX -> 助记词
```

3. 如果弹出 `输入 PIN 码`，输入开发板 PIN。
4. Kern 显示连接二维码。
5. 如果二维码有多片，保持页面显示完整，让手机或浏览器继续扫描；如果有 `显示大码`，扫不动时先点 `显示大码`。
6. 去 OKX 钱包里选择硬件钱包/Keystone 扫这个二维码。

### 用智能卡连接

1. 插好外接供电 USB 读卡器。
2. 把 Satochip/SeedKeeper 智能卡插进读卡器。
3. 建议先跑一次：

```text
设备检查 -> 智能卡
```

4. 确认能识别读卡器、ATR 和 Satochip/SeedKeeper。
5. 在 Kern 首页进入：

```text
连接钱包 -> OKX -> 智能卡
```

6. 屏幕写 `智能卡 PIN` 时，输入智能卡 PIN，不要输入开发板 PIN。
7. Kern 从卡里读取账户信息后显示连接二维码。
8. 去 OKX 钱包里选择硬件钱包/Keystone 扫这个二维码。

### 每个 Web3 钱包在 Kern 上的路径

| 钱包 | 开发板助记词路径 | 智能卡路径 |
|---|---|---|
| OKX Wallet | `连接钱包 -> OKX -> 助记词` | `连接钱包 -> OKX -> 智能卡` |
| Bitget Wallet | `连接钱包 -> Bitget -> 助记词` | `连接钱包 -> Bitget -> 智能卡` |
| imToken | `连接钱包 -> imToken -> 助记词` | `连接钱包 -> imToken -> 智能卡` |
| MetaMask | `连接钱包 -> MetaMask -> 助记词` | `连接钱包 -> MetaMask -> 智能卡` |
| Rabby | `连接钱包 -> Rabby -> 助记词` | `连接钱包 -> Rabby -> 智能卡` |
| TokenPocket | `连接钱包 -> TokenPocket -> 助记词` | `连接钱包 -> TokenPocket -> 智能卡` |
| Keystone | `连接钱包 -> Keystone -> 助记词` | `连接钱包 -> Keystone -> 智能卡` |

`Keystone` 适合其他支持 Keystone/QR 硬件钱包协议的钱包。能选具体钱包名时，优先用钱包名对应的连接码。

## OKX Wallet 连接教程

手机端或插件端按钮名称会随版本变化，看到类似意思即可。

1. 打开 OKX Wallet。
2. 进入添加钱包或账户页面。
3. 选择：

```text
Hardware Wallet
Keystone
```

4. 如果 OKX 提示相机权限，允许相机。
5. 用 OKX 扫描 Kern 上的 OKX 连接二维码。
6. 如果 OKX 让你选择账户，先选第一个常用账户。
7. 不确定派生路径时，不要乱切 Ledger/Legacy 路径；先使用默认账户，之后用小额测试确认地址一致。
8. 连接成功后，OKX 里看到的是观察账户，不能单独花钱，签名仍要 Kern。

常见坑：

| 问题 | 处理 |
|---|---|
| OKX 扫了没反应 | 确认你选的是 OKX 里的 `Hardware Wallet -> Keystone`，不是普通扫一扫 |
| 二维码有多片 | 保持 Kern 页面亮着，按钱包提示继续扫完整 |
| 地址不对 | 检查 Kern 上选的是 `助记词` 还是 `智能卡`，以及 OKX 账户选择是否切了路径 |

## Bitget Wallet 连接教程

1. 打开 Bitget Wallet。
2. 进入钱包管理、添加钱包或添加账户页面。
3. 选择：

```text
Hardware Wallet
Scan QR Code
Keystone
```

4. 用 Bitget 扫描 Kern 上的 Bitget 连接二维码。
5. 如果 Bitget 要先创建一个空的软件钱包才能进入硬件钱包入口，可以创建临时壳钱包，但不要把 Kern 助记词导入进去。
6. 选择要导入的账户。
7. 连接后先收小额测试币，确认地址和链都正确。

常见坑：

| 问题 | 处理 |
|---|---|
| Bitget 找不到 Keystone | 找 `Hardware Wallet`、`硬件钱包`、`扫码连接`，不要走 `Import Mnemonic` |
| BTC 和 ETH 同时出现 | Bitget 的 Keystone 连接码可能同时带 BTC/ETH 观察信息，按你要用的链选择 |
| 签名二维码太密 | 直接看本文 `Web3 签名二维码太密怎么办` |

## MetaMask 连接教程

MetaMask 插件通常走 `Connect Hardware Wallet -> QR-based`。MetaMask Mobile 的入口也可能写 Keystone 或 QR-based。

1. 打开 MetaMask。
2. 进入账户菜单。
3. 选择：

```text
Connect Hardware Wallet
QR-based
Keystone
```

4. 允许浏览器或 App 使用相机。
5. 扫描 Kern 上的 MetaMask 连接二维码。
6. MetaMask 显示账户列表时，选择你要用的账户并解锁。
7. 连接成功后，账户旁边通常会显示硬件钱包或 QR 标识。

常见坑：

| 问题 | 处理 |
|---|---|
| 浏览器没相机权限 | 在浏览器地址栏或系统设置里允许摄像头 |
| 扫码页面打不开 | 换 Chrome/Edge，或用手机端 MetaMask |
| 地址不一致 | 先不要转账，回 Kern 重新确认来源、路径和账户 |

实测注意：

- MetaMask 已用 `crypto-hdkey` 单账户连接码跑通。
- 签名请求使用 `eth-sign-request`，签名回传使用 `eth-signature`。
- MetaMask 的签名请求可能没有 `origin`，回传时不要强行写钱包名。
- 详细金标准见 `docs/WEB3_IMTOKEN_METAMASK_PROVEN_FLOW.zh-CN.md`。

## imToken 连接教程

imToken 已用 Keystone 兼容二维码跑通，连接和签名流程与 MetaMask 类似。

1. 打开 imToken。
2. 进入添加钱包或硬件钱包入口。
3. 选择 Keystone 或二维码硬件钱包。
4. 扫描 Kern 上的 imToken 连接二维码。
5. 账户地址应与 Kern 显示地址一致。
6. DApp 签名时，imToken 会显示动态 `eth-sign-request` 二维码。
7. Kern 签名完成后，imToken 扫描 `eth-signature` 回传二维码完成签名。

实测注意：

- imToken 必须使用单账户 `crypto-hdkey` 连接码。
- 不要使用 `crypto-multi-accounts`。
- request id 必须原样回填。
- recovery id 使用 `0/1`，不要使用 `27/28`。
- 详细金标准见 `docs/WEB3_IMTOKEN_METAMASK_PROVEN_FLOW.zh-CN.md`。

## Rabby 连接教程

Rabby 对 Keystone 支持比较直接，适合 EVM 日常使用。

1. 打开 Rabby 浏览器插件。
2. 进入添加账户或连接硬件钱包。
3. 选择：

```text
Connect Hardware Wallet
Keystone
```

4. 用 Rabby 扫描 Kern 上的 Rabby 连接二维码。
5. 选择账户。
6. Rabby 会把它当成硬件钱包账户管理。

常见坑：

| 问题 | 处理 |
|---|---|
| Rabby 同时打开多个浏览器 | 只保留一个浏览器和一个 Rabby 插件窗口，避免相机或账户冲突 |
| 扫码失败 | 缩放浏览器页面或点 Kern 的大码显示 |
| 交易前提示风险 | 先在 Rabby 和 Kern 两边都核对交易，不懂就拒签 |

## TokenPocket 连接教程

TokenPocket 不同版本的入口名字差异比较大，重点是找能扫描 Keystone/冷钱包/硬件钱包二维码的入口。

1. 打开 TokenPocket。
2. 进入钱包管理、添加钱包或导入钱包页面。
3. 不要选助记词导入。
4. 找类似下面的入口：

```text
硬件钱包
冷钱包
观察钱包
连接其他钱包
Keystone
扫码导入
```

5. 如果页面里能选 Keystone，选择 Keystone。
6. 如果页面里只有冷钱包或观察钱包，选择支持 EVM/离线签名二维码的入口。
7. 用 TokenPocket 扫描 Kern 上的 TokenPocket 连接二维码。
8. 导入账户后，先用小额测试转入和签名。

常见坑：

| 问题 | 处理 |
|---|---|
| 只有 `导入助记词` | 不要继续，返回找硬件钱包/冷钱包入口 |
| 扫普通首页扫一扫不认 | 必须从钱包连接或硬件钱包入口进入扫码 |
| DApp 签名二维码很密 | 用安卓中转 App 或录屏放大再扫 |

## Web3 扫码签名教程

连接钱包只做一次或偶尔做。真正花钱时，每一笔都要走签名流程。

### 手机或插件发起签名

1. 在 OKX、Bitget、imToken、MetaMask、Rabby 或 TokenPocket 里发起转账、授权、Swap 或 DApp 操作。
2. 钱包显示待签名二维码。
3. 不要急着扫，先在手机钱包里看一遍收款地址、金额、链、Gas、DApp 名称。

### Kern 扫码

1. Kern 首页进入：

```text
扫码签名
```

2. 用 Kern 摄像头扫描手机或电脑上的待签名二维码。
3. 如果 Kern 识别为 Web3 请求，会先让你选择：

```text
助记词
智能卡
```

4. 你用开发板助记词连接的钱包，就选 `助记词`。
5. 你用 Satochip 智能卡连接的钱包，就选 `智能卡`。

### Kern 上核对完整交易信息

签名前必须在 Kern 屏幕上核对：

| 信息 | 怎么看 |
|---|---|
| 来源 | `助记词` 或 `智能卡` 是否正确 |
| 钱包 | OKX、Bitget、imToken、MetaMask、Rabby、TokenPocket 是否符合你发起的钱包 |
| 动作 | 转账、消息签名、交易签名、授权等 |
| 链 | Ethereum、BSC、Polygon 等链是否正确 |
| DApp | 是否是你正在使用的网站或 App |
| 路径 | 派生路径是否是你连接时使用的账户 |
| 地址 | 请求签名的地址是否是你的观察账户 |
| 收款地址 | 转账目标是否正确 |
| 金额 | 金额和单位是否符合预期 |
| Gas | Gas Limit、Gas Price、Max Fee 是否离谱 |
| 交易数据 | 如果是合约调用，数据会比较长，不懂就不要签 |
| 原文 | Kern 会显示 `交易原文`、`消息原文` 或 `完整 TypedData` 方便核对 |

如果任何一项看不懂、和手机钱包不一致、或者 DApp 不是你主动打开的，直接返回拒签。

### 输入 PIN 并签名

| 选择来源 | 下一步 |
|---|---|
| 助记词 | 核对后点 `签名` |
| 智能卡 | 核对后点 `输入 PIN`，输入智能卡 PIN 后签名 |

签名完成后，Kern 会显示 `Web3 签名结果` 二维码，通常是 `eth-signature` 格式。

### 手机或插件扫回结果

1. 回到原钱包。
2. 找 `Get Signature`、`扫描签名结果`、`下一步` 或类似按钮。
3. 用原钱包扫描 Kern 上的签名结果二维码。
4. 原钱包识别后，最后再确认广播。

注意：Kern 只是签名，不负责把 Web3 交易广播到链上。广播由 OKX、Bitget、imToken、MetaMask、Rabby 或 TokenPocket 完成。

## Web3 签名二维码太密怎么办

遇到这些情况，不要硬扫：

| 现象 | 处理 |
|---|---|
| Kern 一直扫不出来 | 打开安卓中转 App，把高密度二维码转成低密度二维码 |
| 手机二维码很小 | 放大二维码，亮度调高，保持 10 到 20 厘米距离 |
| 动态二维码跳得很快 | 录屏几秒，再用相册或中转 App 识别 |
| TokenPocket/Bitget 二维码太密 | 优先用安卓中转 App |

安卓中转教程：

```text
docs/ANDROID_RELAY_WALLET_GUIDE.zh-CN.md
```

中转 App 不保存私钥，不签名，只负责把难扫的二维码换成 Kern 更容易扫的大码。

## TypedData 和授权要特别小心

TypedData、EIP-712、Approve、Permit、SetApprovalForAll 这类请求经常不是普通转账。

| 类型 | 风险 |
|---|---|
| `Approve` | 可能授权某个合约花你的代币 |
| `Permit` | 可能不用链上交易就给出授权签名 |
| `SetApprovalForAll` | NFT 场景下可能授权对方转走一整类 NFT |
| `TypedData` | 内容结构复杂，必须看完整原文 |

不懂就拒签。DApp 催你签、客服让你签、空投网站让你签，都先停下来。

## BTC 连接钱包总规则

BTC 钱包连接不是 Web3 的 Keystone 连接码。BTC 连接通常是把 Kern 显示的扩展公钥导入到手机或电脑钱包里，生成只读观察钱包。

| 名称 | 用途 | 地址类型 |
|---|---|---|
| `zpub` | 推荐默认选择 | 原生隔离见证，地址一般以 `bc1q` 开头 |
| `xpub` | 兼容老钱包或旧地址 | 传统地址，常见以 `1` 开头 |
| `ypub` | 嵌套隔离见证 | 地址一般以 `3` 开头，当前主要在智能卡高级公钥页查看 |
| `vpub` | 测试网原生隔离见证 | 测试网 |
| `tpub` | 测试网传统扩展公钥 | 测试网 |
| `upub` | 测试网嵌套隔离见证 | 测试网 |

新手建议：

```text
主网 BTC：优先选 zpub
老地址兼容：才选 xpub
测试网：只在明确测试时用 tpub/upub/vpub
```

扩展公钥不会直接花钱，但会暴露你的收款地址、找零地址和交易历史。不要随便发给别人。

## BTC：在 Kern 上显示 zpub/xpub

### 用开发板助记词导出

1. 确认 Kern 已加载或创建助记词。
2. 首页进入：

```text
连接钱包 -> BTC -> 助记词 -> zpub
```

3. 如果弹出 `输入 PIN 码`，输入开发板 PIN。
4. Kern 显示 `zpub` 二维码和文本。
5. 用手机或电脑 BTC 钱包扫描或手动复制。

如果你的旧钱包只支持传统地址，再用：

```text
连接钱包 -> BTC -> 助记词 -> xpub
```

### 用智能卡导出

1. 插好外接供电读卡器和 Satochip 卡。
2. 首页进入：

```text
连接钱包 -> BTC -> 智能卡 -> zpub
```

3. 屏幕写 `智能卡 PIN` 时，输入智能卡 PIN。
4. Kern 显示智能卡账户的 `zpub` 二维码和文本。

如果要兼容旧地址：

```text
连接钱包 -> BTC -> 智能卡 -> xpub
```

智能卡高级查看入口：

```text
智能卡工具 -> Satochip -> BTC公钥 -> zpub / ypub / xpub / vpub / upub / tpub
```

这个入口主要用于只读检查和高级排障，不等于已经开放 Satochip BTC 卡片 PSBT 生产签名。

## BlueWallet 连接教程

BlueWallet 适合手机上做 BTC 只读观察钱包。

### 导入只读钱包

1. 在 Kern 上显示：

```text
连接钱包 -> BTC -> 助记词 -> zpub
```

或：

```text
连接钱包 -> BTC -> 智能卡 -> zpub
```

2. 打开 BlueWallet。
3. 点击添加钱包。
4. 选择导入钱包。
5. 选择扫描或粘贴。
6. 扫描 Kern 上的 `zpub` 二维码，或手动粘贴 `zpub` 文本。
7. 确认导入的是 watch-only、只读、观察钱包。
8. 进入钱包后，先对比第一个收款地址是否和 Kern 或你预期的一致。

如果 BlueWallet 版本不识别 `zpub`，再尝试 `xpub`，但地址类型会变成旧格式，不要混用。

### 用 BlueWallet 发起 BTC 签名

1. 在 BlueWallet 观察钱包里点击发送。
2. 输入收款地址和金额。
3. 设置矿工费。
4. 生成待签名交易或 PSBT 二维码。
5. Kern 首页进入：

```text
扫码签名
```

6. 用 Kern 扫 BlueWallet 的 PSBT 二维码。
7. 在 Kern 上核对收款地址、金额、找零地址和矿工费。
8. 确认后输入开发板 PIN 并签名。
9. Kern 显示签名后的 PSBT 二维码。
10. BlueWallet 扫回签名结果。
11. BlueWallet 最后广播交易。

如果 BlueWallet 找不到 PSBT/离线签名入口，先用 Sparrow 或 Electrum 验收 BTC 签名流程。

## Sparrow Wallet 连接教程

Sparrow 适合电脑上做 BTC 观察钱包、PSBT、硬件钱包验收。

### 导入 zpub/xpub

1. 在 Kern 上显示 `zpub`，新手优先用：

```text
连接钱包 -> BTC -> 助记词 -> zpub
```

2. 打开 Sparrow。
3. 选择创建新钱包。
4. 钱包类型选择单签。
5. 脚本类型选择 Native SegWit 或 P2WPKH。
6. Keystore 选择 `xPub / Watch Only Wallet`，或在 Keystore 里选择导入扩展公钥。
7. 粘贴 Kern 显示的 `zpub`。
8. 如果 Sparrow 要求 fingerprint 或 derivation path，按钱包默认的 native segwit 账户路径填写；Kern 的 zpub 对应常用账户路径 `m/84'/0'/0'`。
9. 点击 Apply 保存。
10. 对比第一个收款地址是否正确。

如果你导入的是 `xpub`，脚本类型要选 Legacy 或 P2PKH，对应路径通常是 `m/44'/0'/0'`。

### 用 Sparrow 发起 BTC 签名

1. 在 Sparrow 里进入 Send。
2. 输入收款地址、金额和矿工费。
3. 创建交易。
4. 检查交易详情。
5. 导出 PSBT，选择显示二维码或复制 PSBT。
6. Kern 首页进入 `扫码签名`。
7. Kern 扫 Sparrow 的 PSBT 二维码。
8. 在 Kern 上核对所有输出。
9. 签名后，Kern 显示签名 PSBT 二维码。
10. Sparrow 扫回或导入签名 PSBT。
11. Sparrow 广播交易。

## Electrum 连接教程

Electrum 适合电脑或手机上做 BTC 观察钱包。入口名字可能叫 master public key、public/private keys 或 watching-only。

### 导入 zpub/xpub

1. 在 Kern 上显示 `zpub` 或 `xpub`。
2. 打开 Electrum。
3. 选择新建或恢复钱包。
4. 选择 Standard wallet。
5. 选择使用主公钥、Use a master key，或 Use public or private keys。
6. 粘贴 Kern 显示的 `zpub` 或 `xpub`。
7. Electrum 创建 watch-only 钱包。
8. 设置一个本地钱包文件密码。
9. 对比第一个收款地址。

如果 Electrum 不接受某种扩展公钥前缀，换用 `xpub`，或在 Sparrow 中先确认脚本类型和路径。

### 用 Electrum 发起 BTC 签名

1. 在 Electrum 观察钱包里进入 Send。
2. 输入收款地址和金额。
3. 点击 Preview 预览交易。
4. 导出 unsigned transaction 或 PSBT。
5. 如果有二维码选项，选择给硬件设备使用并包含 xpub 信息。
6. Kern 首页进入 `扫码签名`。
7. Kern 扫 Electrum 的 PSBT 二维码。
8. 在 Kern 上核对输出、找零和矿工费。
9. 签名后，Electrum 扫回或导入签名交易。
10. Electrum 广播。

## Specter Desktop 连接教程

Specter 适合已经有 Bitcoin Core 或节点环境的用户。新手如果只是想快速验收，优先用 BlueWallet、Sparrow 或 Electrum。

### 导入观察设备

1. 在 Kern 上显示 `zpub`。
2. 打开 Specter Desktop。
3. 确认已连接 Bitcoin Core 或可用后端。
4. 添加新设备。
5. 选择 Keystone、Generic、Other 或支持手动扩展公钥的设备类型。
6. 填入 `zpub`、fingerprint 和 derivation path。
7. 如果是 native segwit 单签，路径通常使用 `m/84'/0'/0'`。
8. 用这个设备创建单签 watch-only 钱包。
9. 对比收款地址。

### 用 Specter 发起 BTC 签名

1. 在 Specter 钱包里创建交易。
2. 导出 PSBT。
3. 如果能显示二维码，用二维码给 Kern 扫；如果只能导出文件，当前 Kern 主要按扫码流程验收。
4. Kern 签名后，把签名 PSBT 导回 Specter。
5. Specter 广播交易。

## Nunchuk 连接教程

Nunchuk 版本差异比较明显。它更常用于多签和硬件钱包协作，新手先用 BlueWallet/Sparrow/Electrum 会更稳。

如果你的 Nunchuk 版本支持导入扩展公钥或扫描钱包配置：

1. 在 Kern 上显示 `zpub`。
2. 打开 Nunchuk。
3. 添加 Key 或添加 Wallet。
4. 选择导入已有公钥、scan QR、import xpub，或 watch-only 相关入口。
5. 扫描 Kern 的 `zpub`。
6. 用这个 key 创建单签钱包，或作为多签其中一个 cosigner。
7. 对比收款地址。

如果你的版本只有导入助记词或私钥，不要使用它连接 Kern，换 Sparrow 或 BlueWallet。

## BTC 签名前必须核对什么

Kern 扫到 PSBT 后，不要只看手机钱包。必须在 Kern 屏幕上核对。

| 信息 | 为什么重要 |
|---|---|
| 收款地址 | 防止剪贴板或钱包界面被替换 |
| 收款金额 | 防止金额被改 |
| 找零地址 | 找零应该回到你自己的钱包路径 |
| 矿工费 | 防止异常高费率 |
| 输入数量 | 大量输入可能暴露隐私或导致手续费很高 |
| 钱包来源 | 确认是当前加载助记词对应的钱包 |

看不懂找零地址时，先不要签。可以用 Sparrow/Electrum 的交易详情和 Kern 的显示互相对照。

## 常见问题

### 手机钱包问我要助记词怎么办

不要输入。返回上一级，找 `硬件钱包`、`Keystone`、`QR-based`、`冷钱包` 或 `观察钱包`。

### Web3 钱包扫 Kern 二维码不认

按顺序检查：

1. 是否从钱包的硬件钱包/Keystone 入口扫码。
2. Kern 上选的钱包名是否和手机钱包一致。
3. 二维码是否完整显示，没有被边框挡住。
4. 如果是 OKX 动态连接码，是否扫完所有片。
5. 手机相机权限是否打开。
6. 换 `显示大码` 或调高亮度。

### 签名时只能选智能卡，不能选助记词

当前统一 Web3 扫码入口应该显示 `助记词` 和 `智能卡` 两个按钮。如果只看到智能卡：

1. 确认你从首页 `扫码签名` 进入，而不是从智能卡工具里的专用入口进入。
2. 确认固件已经包含最新修改。
3. 确认已经加载可签名助记词，不是临时助记词。
4. 仍不对时，先不要签名，把屏幕拍照和固件版本给开发者排查。

### 为什么连接后手机钱包能看到资产，却不能直接转出

这是正常的。手机钱包是观察钱包，只能发起交易、显示余额和广播。真正签名必须由 Kern 开发板或智能卡完成。

### BTC 导入后地址不一样

常见原因：

| 原因 | 处理 |
|---|---|
| 用了 `xpub` 但钱包按 native segwit 解析 | 改用 `zpub` 或调整脚本类型 |
| 用了 `zpub` 但钱包按 legacy 解析 | 在钱包里选择 Native SegWit/P2WPKH |
| 助记词来源不一致 | 确认 Kern 选的是 `助记词` 还是 `智能卡` |
| 账户号不同 | 新手先用账户 0，对应常见路径 `m/84'/0'/0'` |
| 主网/测试网混了 | 主网用 zpub/xpub，测试网用 vpub/tpub/upub |

### 智能卡 PIN 输错了怎么办

看到剩余次数减少就立刻停下，不要猜。智能卡 PIN 错误次数归零会锁卡。详细处理看：

```text
docs/SECURITY_PIN_GUIDE.zh-CN.md
```

### 开发板 PIN 输错了怎么办

开发板 PIN 连续错 3 次会清除本次会话并关机或重启。停下来确认 PIN，不要连续尝试。详细处理看：

```text
docs/SECURITY_PIN_GUIDE.zh-CN.md
```

## 新手验收顺序

第一次别直接上主网大额资金。按这个顺序最稳：

1. 只用测试助记词或小额测试资金。
2. 用 `连接钱包 -> OKX -> 助记词` 连接 OKX。
3. 发一笔小额 Web3 转账，Kern 核对后签名。
4. 用 `连接钱包 -> BTC -> 助记词 -> zpub` 导入 BlueWallet 或 Sparrow。
5. 发一笔小额 BTC 测试交易，Kern 扫 PSBT 后签名。
6. 插智能卡，跑 `设备检查 -> 智能卡`。
7. 用 `连接钱包 -> OKX -> 智能卡` 连接 Web3。
8. 只用测试资金做一笔 Satochip Web3 签名。
9. 确认每一步都能解释清楚以后，再考虑更大金额。
