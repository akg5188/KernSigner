# KernSigner 小白照抄完整使用手册

日期：2026-05-22

这份手册给第一次拿到 KernSigner 固件的人看。你可以把它当成“照着点屏幕”的主手册：先按顺序完成首刷、开机、安全初始化、创建测试助记词、备份、连接钱包、扫码签名，再按需要查看智能卡、直接扫码排障和生产边界。

当前固件是测试资金验收版。没有完成商业发布门禁、真机验收和独立审计前，不要导入真实助记词，不要存放真实资产，不要把它宣传成已审计生产钱包。

## 0. 先看一句话

KernSigner 开发板负责离线保存或临时加载密钥、显示地址、扫码签名；手机或电脑钱包只负责联网查看余额、构造交易和广播；私钥、助记词、智能卡 PIN 不应该进入手机钱包、浏览器插件、联网网页或聊天软件。

## 1. 你需要准备什么

| 物品 | 用途 | 新手注意 |
| --- | --- | --- |
| Waveshare ESP32-P4-WiFi6-Touch-LCD-4.3 | 运行 KernSigner 固件 | 只支持 `wave_43` 这块 4.3 寸板 |
| USB 数据线 | 供电、刷机、串口日志 | 刷机走 USB-to-UART/下载口 |
| OV5647 摄像头 | 扫二维码、相机熵 | 首次使用要手动调焦；没有摄像头就不能扫码 |
| 纸和笔 | 抄助记词、PIN、验收信息 | 助记词和 PIN/PUK 分开记录、分开保存；不拍照、不发网盘 |
| 手机或电脑钱包 | 导入观察钱包、发起交易 | 不导入 KernSigner 助记词 |
| 可选：Satochip/SeedKeeper | 智能卡签名或秘密保存 | 必须带供电 OTG/Hub |
| 可选：电脑放大二维码 | 把太密或太小的二维码放大显示 | 只用于辅助显示，不保存敏感材料 |

## 2. 先认清接口

只按下面这样用：

```text
USB-to-UART / 下载口 -> 接电脑，用于刷机和串口日志
USB OTG 口 -> 接外设，用于带供电 Hub、读卡器、智能卡
摄像头 -> 扫二维码和相机随机熵
触摸屏 -> 日常全部操作
```

不要这样做：

- 不要把智能卡读卡器插到下载口。
- 不要用无供电 OTG 线直接带 ACR39U-NF 读卡器。
- 不要把别的 Waveshare 屏幕板当成已支持硬件。
- 如果使用仓库里的 3D 打印外壳草模，不要以为仍能方便插 SD 卡或按侧边实体键；当前草模封住了 TF/SD 和 POWER/BOOT/RESET 侧边孔。

智能卡正确接法：

```text
ESP32-P4 OTG 口
-> 带供电 OTG 转接线 / 真正外接供电 Hub
-> ACR39U-NF 读卡器
-> Satochip 或 SeedKeeper 智能卡
```

## 3. 下载和刷机

如果你已经拿到刷好固件的设备，可以跳到“第一次开机”。如果你要自己刷固件，按下面做。

### 3.1 选择固件

| 情况 | 刷哪个文件 | 地址 |
| --- | --- | --- |
| 新板、空白板、状态不明的板 | `kernsigner-wave43-0.0.7-rc1-untested-full.bin` | `0x0` |
| 已经运行本项目，只升级应用 | `kernsigner-wave43-0.0.7-rc1-untested-app.bin` | `0x20000` |

固件目录：

```text
firmware/wave_43/
```

### 3.2 校验固件

Linux/macOS：

```bash
cd firmware/wave_43
sha256sum -c SHA256SUMS.txt
```

Windows PowerShell：

```powershell
cd firmware\wave_43
Get-FileHash .\kernsigner-wave43-0.0.7-rc1-untested-full.bin -Algorithm SHA256
```

如果校验失败，停止，不要刷机。重新下载或重新拷贝固件。

Windows 上要把 `Get-FileHash` 输出的哈希和 `SHA256SUMS.txt` 里对应文件的哈希逐字符比对，一位不同都不要刷。

### 3.3 Linux 一键刷完整固件

先把开发板下载口接电脑。假设串口是 `/dev/ttyACM0`：

```bash
cd firmware/wave_43
./flash_wave43_linux.sh /dev/ttyACM0
```

如果串口不是 `/dev/ttyACM0`，先查：

```bash
ls /dev/ttyACM*
ls /dev/ttyUSB*
```

### 3.4 Windows 一键刷完整固件

先在设备管理器里看串口，例如 `COM3`，然后运行：

```bat
cd firmware\wave_43
flash_wave43_windows.bat COM3
```

`COM3` 换成你的实际串口。

### 3.5 刷完最低检查

1. 设备自动重启或手动重启。
2. 屏幕亮起。
3. 触摸能点。
4. 能进入设置 PIN 或首页。
5. 能进入 `自检 -> 系统` 或 `自检 -> 外设`。
6. 能打开相机或扫码页面。
7. 用普通二维码检查摄像头焦距；如果不灵敏，捏住镜头外圈小幅旋转到二维码边缘最清楚。
8. 如果接智能卡，先用带电 OTG，再进入 `自检 -> 智能卡`。

刷完黑屏、花屏、触摸不准、开不了机，先看 [故障排查照抄手册](故障排查照抄手册.zh-CN.md)。

## 4. 第一次开机和开发板 PIN

开发板 PIN 是 KernSigner 这块设备自己的 PIN。它不是智能卡 PIN。

第一次开机：

1. 插 USB 给开发板供电。
2. 等屏幕亮起。
3. 如果看到 `设置 PIN 码`，输入一个新的开发板 PIN。
4. PIN 至少 6 位，不要用 `123456`、生日、手机号。
5. 如果看到 `输入 PIN 码`，输入开发板 PIN。
6. 进入首页后，不要急着导入真钱包，先做测试流程。

PIN 判断口诀：

| 屏幕文字 | 输入什么 |
| --- | --- |
| `设置 PIN 码` | 新的开发板 PIN |
| `输入 PIN 码` | 开发板 PIN |
| `智能卡 PIN` | Satochip 或 SeedKeeper 卡 PIN |
| `PUK` | 智能卡 PUK |

不确定就返回上一页，不要猜。开发板 PIN 连续错 3 次会清除本次会话并关机或重启；智能卡 PIN/PUK 输错会消耗卡片自己的剩余次数。

完整 PIN 说明看 [SECURITY_PIN_GUIDE.zh-CN.md](SECURITY_PIN_GUIDE.zh-CN.md)。

## 5. 首页怎么选

首页常见入口：

| 首页入口 | 你想做什么 | 去哪里 |
| --- | --- | --- |
| `扫码签名` | 扫 PSBT、Web3 请求、消息或二维码数据 | 签名前核对完整信息 |
| `连接钱包` | 生成 OKX、Bitget、MetaMask、Rabby、TokenPocket、BTC 观察钱包二维码 | 不导出私钥 |
| `助记词` | 创建、导入、备份、写卡、密语、高级工具 | 所有秘密材料都在这里 |
| `智能卡` | Satochip/SeedKeeper 信息、地址、写入、PIN、重置 | 先保证带电 OTG |
| `设置` | PIN、亮度、相机、钱包、安全、关于 | 日常设备设置 |
| `自检` | 系统、安全、交付、智能卡、外设 | 首刷和排错先用 |
| `功能` | 查看功能说明或状态 | 用来确认入口能力 |
| `工具` | 创建普通二维码、扫码、文件工具 | 不一定是签名流程 |

完整菜单树看 [功能菜单逐项索引](功能菜单逐项索引.zh-CN.md)。

## 6. 创建测试助记词

第一次练习只用测试助记词。

路径：

```text
首页 -> 助记词 -> 创建
```

创建方式：

| 方式 | 适合谁 | 照做方法 |
| --- | --- | --- |
| `十六进制` | 想最容易验证 | 12 词输入 32 个十六进制字符，24 词输入 64 个 |
| `骰子` | 有六面骰 | 每次输入 `1-6` |
| `抛硬币` | 有硬币 | 正面记 `1`、反面记 `0`，从头到尾一致 |
| `D20` | 有二十面骰 | 每次输入 `1-20` |
| `扑克牌` | 有实体牌 | 洗牌后输入 `AH QS 9D TC` 这类牌面 |
| `手动` | 已有测试单词 | 输入 BIP39 英文单词 |
| `相机` | 想快速取熵 | 对复杂画面拍摄 |
| `智能卡` | 用 SeedKeeper 随机数 | 先接带电 OTG 和卡 |

最简单测试向量：

```text
00000000000000000000000000000000
```

预期 12 词：

```text
abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon about
```

这只是公开测试助记词，绝不能放真实资产。

详细验证看 [MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md](MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md)。

## 7. 导入已有助记词

路径：

```text
首页 -> 助记词 -> 导入
```

导入方式：

| 方式 | 用途 | 注意 |
| --- | --- | --- |
| `二维码` | 扫 SeedQR、Compact SeedQR 或支持的助记词 QR | 二维码等同助记词 |
| `手动` | 输入 12/15/18/21/24 个英文单词 | 单词和顺序必须完全对 |
| `序号` | 输入 BIP39 编号 | 注意编号口径 |
| `智能卡` | 从 SeedKeeper 导入 | 输入智能卡 PIN |
| `存储卡` | 从 SD/文件导入 | 取决于硬件和外壳是否可用 |
| `加密` | 导入 KEF 加密备份 | 密码丢了无法恢复 |
| `钢板` | 从钢板序号恢复 | 恢复后核对指纹 |
| `点阵` | Tinyseed/1248/Stackbit 类恢复 | 注意每词位点 |

导入成功后必须核对：

1. 钱包指纹。
2. 第一个收款地址。
3. 扩展公钥。
4. 网络：主网还是测试网。
5. 派生路径。
6. 是否使用 BIP39 密语。

只看“导入成功”不够。地址不一致就停止。

## 8. 备份和恢复演练

创建或导入助记词后，先备份，再收款。

常见路径：

```text
首页 -> 助记词 -> 本机助记词 -> 备份
```

或在已加载钱包菜单中进入：

```text
本机助记词 -> 备份
```

推荐新手最低备份：

1. 打开明文助记词或序号页面。
2. 用纸按顺序抄写单词和序号。
3. 抄完逐词核对。
4. 清除会话。
5. 重新用 `导入 -> 手动` 导入。
6. 核对钱包指纹和第一个地址。

备份方式选择：

| 备份方式 | 怎么用 | 风险 |
| --- | --- | --- |
| `序号` / 明文单词 | 最基础，必须会 | 被看到就能偷钱 |
| `原始熵` | BIP39 精确验证和高级备份 | 泄露等同助记词 |
| `二维码` | 临时快速迁移或恢复演练 | 不拍照、不存手机 |
| `加密` / KEF | 额外文件备份 | 文件和密码都要保管 |
| `点阵板` / `钢板` / `1248` | 金属长期备份 | 刻错要靠恢复演练发现 |
| `写卡` | 写入 Satochip 或 SeedKeeper，入口在 `首页 -> 助记词 -> 写卡`，不是 `备份` 子菜单 | 卡 PIN/PUK 管理复杂，写错或覆盖很难恢复 |

完整备份说明看 [BACKUP_AND_RECOVERY_GUIDE.zh-CN.md](BACKUP_AND_RECOVERY_GUIDE.zh-CN.md)。

## 9. 使用 BIP39 密语

BIP39 密语也叫 passphrase 或密码短语。它不是登录密码，而是参与种子派生的额外要素，会改变所有钱包指纹和地址。

路径：

```text
首页 -> 助记词 -> 密语
```

规则：

| 助记词 | 密语 | 结果 |
| --- | --- | --- |
| 同一组助记词 | 空 | 钱包 A |
| 同一组助记词 | `abc` | 钱包 B |
| 同一组助记词 | `ABC` | 钱包 C |

用了密语后，备份必须同时保存“助记词”和“密语”。丢任何一个都恢复不出同一个钱包。

## 10. 查看地址和公钥

常见入口：

```text
首页 -> 助记词 -> 本机助记词 -> 公钥
首页 -> 助记词 -> 本机助记词 -> 派生地址
首页 -> 连接钱包 -> 派生地址
```

你应该核对：

| 项目 | 为什么 |
| --- | --- |
| 网络 | 主网和测试网地址不同 |
| 地址类型 | legacy、nested SegWit、native SegWit、Taproot 不同 |
| 派生路径 | `m/84'/0'/0'` 和 `m/49'/0'/0'` 不是一个钱包视图 |
| 账户号 | account 0、account 1 不同 |
| 密语 | 有无密语完全不同 |

扩展公钥不能花钱，但会暴露你的地址和交易历史。不要公开发给无关人员。

## 11. 连接 BTC 观察钱包

推荐用观察钱包收款和构造 PSBT，KernSigner 只离线签名。

KernSigner 上显示公钥：

```text
首页 -> 连接钱包 -> BTC -> 助记词 -> zpub
```

或智能卡来源：

```text
首页 -> 连接钱包 -> BTC -> 智能卡 -> zpub
```

如果钱包需要 legacy 观察公钥，可以选：

```text
首页 -> 连接钱包 -> BTC -> 助记词 -> xpub
```

使用步骤：

1. KernSigner 显示 `zpub` 或 `xpub` 二维码。
2. 手机或电脑钱包选择导入观察钱包。
3. 扫描 KernSigner 二维码。
4. 钱包生成第一个收款地址。
5. 回 KernSigner 上查看同路径地址。
6. 两边地址一致后，只做小额测试收款。

地址不一致，不要收款。

## 12. BTC PSBT 签名

推荐主线：本机助记词签名。Satochip BTC PSBT 卡签名当前不要作为生产可用能力宣传。

步骤：

1. 观察钱包创建交易。
2. 钱包显示 PSBT 二维码。
3. KernSigner 进入：

```text
首页 -> 扫码签名
```

4. 用 KernSigner 摄像头扫描 PSBT。
5. 如果是多片二维码，从第一帧开始扫完整。
6. 在 KernSigner 上核对收款地址、金额、找零地址、手续费、网络。
7. 如果弹出 `输入 PIN 码`，输入开发板 PIN。
8. 最后确认签名。
9. KernSigner 显示签名结果二维码。
10. 观察钱包扫回签名 PSBT 或交易。
11. 在观察钱包里广播。

遇到下面任一情况，取消：

- 收款地址不对。
- 金额不对。
- 找零地址不认识。
- 手续费异常。
- 网络不一致。
- 页面看不懂。

## 13. 连接 Web3 钱包

支持入口包括：

```text
OKX / Bitget / MetaMask / Rabby / TokenPocket
```

KernSigner 生成连接二维码：

```text
首页 -> 连接钱包 -> OKX -> 助记词
首页 -> 连接钱包 -> OKX -> 智能卡
```

把 `OKX` 换成 `Bitget`、`MetaMask`、`Rabby`、`TokenPocket` 即可。

手机或浏览器钱包里必须找类似入口：

```text
硬件钱包
Connect Hardware Wallet
Keystone
QR-based
扫码连接冷钱包
```

不要选择：

```text
导入助记词
导入私钥
Import seed phrase
Import private key
```

连接后，手机钱包里只是观察账户或硬件账户。真正签名仍在 KernSigner 或智能卡里完成。

## 14. Web3 扫码签名

步骤：

1. 在 OKX、Bitget、MetaMask、Rabby 或 TokenPocket 里发起测试转账、授权或 DApp 操作。
2. 钱包显示待签名二维码。
3. KernSigner 进入：

```text
首页 -> 扫码签名
```

4. KernSigner 扫手机或电脑上的签名请求二维码。
5. 页面让你选择来源时，选你连接时用的来源：

```text
助记词
智能卡
```

6. 用助记词来源时，按页面输入开发板 PIN 或确认。
7. 用智能卡来源时，看到 `智能卡 PIN` 才输入 Satochip PIN。
8. 在 KernSigner 上核对链、from 地址、to 地址、金额、Gas、DApp、合约、授权。
9. 看不懂就取消。
10. 确认签名后，KernSigner 显示签名结果二维码。
11. 手机钱包扫回结果。
12. 手机钱包最终确认广播。

Web3 特别危险的是授权和合约调用。只要 KernSigner 上不能清楚显示或你看不懂，就不要签。

## 15. 二维码太密怎么办

照做：

1. 首次使用 ESP32-P4 4.3 开发板时，先用普通黑白二维码调 OV5647 摄像头焦距：保持 10 到 20 厘米，捏住镜头外圈小幅旋转到最清楚。
2. 手机亮度调高。
3. 二维码完整显示，不要被边框、按钮、刘海遮住。
4. KernSigner 和屏幕保持 10 到 20 厘米距离。
5. 避免反光，不要斜着扫。
6. 如果手机二维码太小，截图或投屏到电脑，把二维码放大到屏幕中央。
7. 如果是动态二维码，从第一帧开始保持几秒，不要频繁移动。
8. OKX 圆点动态码完全没反应时，先按 [扫码与摄像头排障手册](QR_CAMERA_TROUBLESHOOTING.zh-CN.md) 查焦距、图像解码和 UR 分片。
9. 仍然失败时，保存清晰照片或短录屏给开发者用电脑端 `zbarimg` / `components/zbar_qr/test` 验证。

## 16. 使用 Satochip

Satochip 主要用于智能卡签名和账户读取。当前可按测试卡流程使用，不能当作已审计生产能力宣传。

第一次使用空白 Satochip：

1. 按正确链路接好带电 OTG、读卡器和 Satochip。
2. 进入：

```text
首页 -> 自检 -> 智能卡
```

3. 确认能看到读卡器、ATR、Satochip。
4. 进入：

```text
首页 -> 智能卡 -> Satochip -> 维护 -> 设置PIN
```

5. 设置 Satochip PIN，并记录好。
6. 准备好本机测试助记词，并确认已经有纸质备份。
7. 确认这是空白测试卡，或确认卡内已有种子可以被覆盖；如果不确定，立刻停止，不要写卡。
8. 写入 Satochip：

```text
首页 -> 助记词 -> 写卡 -> Satochip
```

或：

```text
首页 -> 智能卡 -> Satochip -> 写入
```

9. 写入后用 `地址`、`BTC公钥` 或 `连接钱包 -> 钱包名 -> 智能卡` 验证地址。

Satochip 常用路径：

| 目的 | 路径 |
| --- | --- |
| 看卡信息 | `首页 -> 智能卡 -> Satochip -> 信息` |
| 看 EVM/BTC 路径地址 | `首页 -> 智能卡 -> Satochip -> 地址` |
| 导出 BTC 观察公钥 | `首页 -> 智能卡 -> Satochip -> BTC公钥` |
| Web3 连接 | `首页 -> 连接钱包 -> 钱包名 -> 智能卡` |
| Web3 签名 | `首页 -> 扫码签名 -> 扫请求 -> 智能卡` |
| 改 PIN/解锁/重置 | `首页 -> 智能卡 -> Satochip -> 维护` |

不要连续猜卡 PIN。剩余次数只剩 1 次时停止。

## 17. 使用 SeedKeeper

SeedKeeper 主要用于保存助记词、密码、描述符等秘密。它不是 Satochip 的另一个名字，也不是直接 Web3 签名卡；从 SeedKeeper 导入本机后，后续签名来源按 `助记词` 理解。

空白 SeedKeeper 第一次使用：

1. 接好带电 OTG、读卡器和 SeedKeeper。
2. 进入：

```text
首页 -> 自检 -> 智能卡
```

3. 确认能识别卡。
4. 进入：

```text
首页 -> 智能卡 -> SeedKeeper -> 设置PIN
```

5. 设置 SeedKeeper PIN。
6. 写入当前助记词：

```text
首页 -> 助记词 -> 写卡 -> SeedKeeper
```

7. 先列出条目：

```text
首页 -> 智能卡 -> SeedKeeper -> 列表
```

8. 如果要查看卡内助记词，才进入 `查看`。这会暴露秘密，只能在离线、无遮挡摄像头、无旁人、无截图录屏的环境下做。
9. 导入本机后，核对钱包指纹和第一个地址。

SeedKeeper 常用路径：

| 目的 | 路径 |
| --- | --- |
| 看状态 | `首页 -> 智能卡 -> SeedKeeper -> 状态` |
| 设置 PIN | `首页 -> 智能卡 -> SeedKeeper -> 设置PIN` |
| 改 PIN | `首页 -> 智能卡 -> SeedKeeper -> 改PIN` |
| 看空间 | `首页 -> 智能卡 -> SeedKeeper -> 剩余空间` |
| 列条目 | `首页 -> 智能卡 -> SeedKeeper -> 列表` |
| 创建助记词 | `首页 -> 智能卡 -> SeedKeeper -> 创建` |
| 写入当前助记词 | `首页 -> 智能卡 -> SeedKeeper -> 写入` |
| 查看卡内助记词 | `首页 -> 智能卡 -> SeedKeeper -> 查看` |
| 从卡导入本机 | `首页 -> 智能卡 -> SeedKeeper -> 导入` |
| 存密码 | `首页 -> 智能卡 -> SeedKeeper -> 存密码` |
| 存描述符 | `首页 -> 智能卡 -> SeedKeeper -> 存描述符` |
| 读描述符 | `首页 -> 智能卡 -> SeedKeeper -> 读描述符` |

SeedKeeper 重置是高风险操作，会清空卡内所有秘密。只有确认已经有纸质或其他离线备份、并且这就是要清空的测试卡时才继续。新版 SeedKeeper 不是旧 `B0 FF`，要走错 PIN/错 PUK 到 `FF00` 的流程。详细步骤看 [SMARTCARD_SATOCHIP_SEEDKEEPER_OPERATION_GUIDE.zh-CN.md](SMARTCARD_SATOCHIP_SEEDKEEPER_OPERATION_GUIDE.zh-CN.md)。

## 18. 设置和自检

设置路径：

```text
首页 -> 设置
```

常用项目：

| 项目 | 用途 |
| --- | --- |
| `PIN` | 修改开发板 PIN |
| `亮度` | 调屏幕亮度 |
| `相机` | 相机相关设置 |
| `钱包` | 钱包显示或派生相关设置 |
| `安全` | 安全相关状态或选项 |
| `关于` | 查看版本和项目信息 |

自检路径：

```text
首页 -> 自检
```

常用项目：

| 项目 | 用途 |
| --- | --- |
| `系统` | 看系统状态 |
| `安全` | 看安全配置状态 |
| `交付` | 看交付或版本检查 |
| `智能卡` | 检测读卡器、ATR、卡类型 |
| `外设` | 触摸、相机、存储卡、供电测试 |

首刷后至少跑：屏幕、触摸、亮度、相机、扫码、智能卡检测。

## 19. 日常安全退出

每次用完：

1. 回首页。
2. 选择清除会话，或关机/断电。
3. 拔掉 USB。
4. 收好纸质备份。
5. 检查手机相册、截图、剪贴板里没有助记词。
6. 智能卡拔出后单独保存。

设备会自动锁定，但不要完全依赖自动锁定。

## 20. 小白第一次完整验收顺序

按这个顺序做一遍：

1. 确认硬件是 `Waveshare ESP32-P4-WiFi6-Touch-LCD-4.3`。
2. 校验固件 SHA256。
3. 刷 `full.bin`。
4. 开机设置开发板 PIN。
5. 进入首页。
6. 跑 `自检 -> 系统`。
7. 跑 `自检 -> 外设 -> 触摸`。
8. 跑 `自检 -> 外设 -> 相机`。
9. 用 `十六进制` 创建测试助记词。
10. 查看原始熵，用离线 BIP39 工具验证。
11. 纸上备份助记词。
12. 清除会话。
13. 手动导入助记词。
14. 核对指纹和地址。
15. 连接 BTC 观察钱包，导出 `zpub`。
16. 做一次测试 PSBT 签名。
17. 连接一个 Web3 钱包，例如 OKX 或 Bitget。
18. 做一笔小额测试签名。
19. 如果用智能卡，接带电 OTG 后跑 `自检 -> 智能卡`。
20. 用测试卡跑 Satochip 或 SeedKeeper 流程。
21. 保存版本号、固件 SHA256、测试结果和问题截图。

如果是刚刷完固件的首刷验收，再按这个最小检查单补一遍：

| 顺序 | 检查项 | 通过标准 |
| ---: | --- | --- |
| 1 | 开机 | 能进启动页或首页，串口没有持续重启、panic、assert、watchdog |
| 2 | 触摸 | 主菜单、返回上级、回首页都能正常点 |
| 3 | 亮度 | 调亮度有可见变化，不会黑屏或触摸失效 |
| 4 | 相机预览 | 画面可见、方向合理、能稳定返回 |
| 5 | 二维码分类 | 普通文本、URL、公开收款 URI 能识别；高风险内容只用测试数据 |
| 6 | 存储卡 | FAT32 卡读写、只读浏览、统计和安全过滤正常；如果外壳挡住卡槽，就记录为未测 |
| 7 | 关于页 | 版本和中文信息能显示并返回 |
| 8 | 自检/交付页 | 能看到固件版本、硬件快照和功能边界 |
| 9 | 钱包流程 | 只用测试助记词、测试 PSBT、测试 Web3 请求验证加载、导入、地址、备份、签名、取消和导出 |

交付给别人前至少收集：

```text
固件版本：
构建时间：
源码 commit：
固件 SHA256：
boot.log：
验收照片/截图：
失败项：
复现步骤：
```

生产候选还要附 `PRODUCTION_CHECK.txt`、`MANUFACTURING_READINESS.txt`、签名镜像和烧录记录。`prodcheck` 没过之前，不能标记为商业生产版。

## 21. 什么时候立刻停止

遇到这些情况先停：

- 开发板 PIN 只剩最后一次机会。
- 智能卡 PIN 或 PUK 只剩最后一次机会。
- 手机钱包地址和 KernSigner 地址不一致。
- 签名页面显示的金额、合约、授权看不懂。
- 刷机校验失败。
- 不确定刷的是测试版还是生产版。
- 智能卡读卡器灯只闪一下。
- 页面反复崩溃或重启。

停止后先记录页面标题、错误原文、你刚点了什么、固件版本、串口日志，再继续排查。
