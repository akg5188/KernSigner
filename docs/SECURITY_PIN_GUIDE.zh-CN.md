# Kern PIN 与安全操作指南

日期：2026-05-21

这份文档给第一次使用 Kern 的人看。目标只有一个：看到屏幕上的 PIN 输入框时，知道应该输入哪个 PIN、输错会怎样、什么时候应该停下来。

当前固件仍是测试资金验收版，不是已经审计的商业生产钱包。不要把真实大额资产只放在这台设备或这张智能卡里。

## 先看这 30 秒

Kern 里有两类 PIN，不能混用。

| 名称 | 保护谁 | 什么时候输入 | 输错影响 |
|---|---|---|---|
| 开发板 PIN | Kern 这块开发板、本机助记词、本机签名、备份入口 | 开机解锁、进入本机敏感功能、修改开发板 PIN | 计入开发板错误次数。连续错 3 次会清除本次会话并关机或重启 |
| 智能卡 PIN | 插在读卡器里的 Satochip/SeedKeeper 智能卡 | 页面写着 `智能卡 PIN`、Satochip 签名、读卡地址/公钥、卡片维护 | 计入智能卡自己的错误次数。剩余次数归零会锁卡 |
| 智能卡 PUK | 智能卡 PIN 的救援码 | 页面写着 `PUK` 或 `解锁` | 计入智能卡 PUK 错误次数。PUK 也用完会非常麻烦，可能需要重置卡或找卡片官方工具 |

最简单的判断方法：

| 屏幕文字 | 输入什么 |
|---|---|
| `输入 PIN 码` | 开发板 PIN |
| `设置 PIN 码` | 新的开发板 PIN |
| `PIN 设置` / `修改 PIN` | 先输入当前开发板 PIN，再输入新的开发板 PIN |
| `智能卡 PIN` | 智能卡 PIN，不是开发板 PIN |
| `旧 PIN` / `新 PIN` / `PIN 编号`，位置在 Satochip 或 SeedKeeper 维护页 | 智能卡旧 PIN、新 PIN，普通用户 `PIN 编号` 填 `0` |
| `PUK` / `解锁` | 智能卡 PUK，不是普通 PIN |

如果不确定是哪一个 PIN，不要猜。先返回上一页，看标题和所在菜单。

## 开发板 PIN 是什么

开发板 PIN 是 Kern 设备自己的 PIN。它和智能卡没有关系。

开发板 PIN 用来保护：

| 功能 | 说明 |
|---|---|
| 开机解锁 | 设备启动后必须先解锁 |
| 本机助记词 | 保护已经加载到 Kern 里的助记词和会话 |
| 本机备份 | 查看助记词、二维码、熵、钢板/点阵备份前会要求开发板 PIN |
| 本机地址/公钥 | 查看地址、扩展公钥等敏感派生信息前会要求开发板 PIN |
| 本机 BTC 签名 | PSBT 或消息签名前会要求开发板 PIN，并有最终确认 |
| 修改开发板 PIN | 进入 `设置 -> PIN` 前会先验证当前开发板 PIN |

开发板 PIN 长度：

| 项目 | 当前规则 |
|---|---|
| 最短 | 6 个字符 |
| 最长 | 16 个字符 |
| 字符 | 可以用数字、字母和符号 |

建议使用不容易被猜到的 PIN，不要用生日、手机号、`123456`、`111111`。

## 第一次设置开发板 PIN

第一次启动设备时，如果还没有开发板 PIN，设备会自动进入设置 PIN 页面。

操作步骤：

1. 看到 `设置 PIN 码`。
2. 输入你要设置的新开发板 PIN。
3. 按键盘上的确认键。
4. 如果固件提示 `正在处理...`，等待完成。
5. 设置完成后会进入主界面。

注意：

| 情况 | 怎么处理 |
|---|---|
| 提示 `PIN 码至少需要 6 位` | 重新输入至少 6 位 |
| 输入后没有进入主界面 | 等待处理完成，不要连续乱点 |
| 忘记刚设置的 PIN | 不能从设备里找回，只能清理设备后用助记词备份恢复 |

当前构建的开发板 PIN 设置流程是输入新 PIN 后直接保存。后续版本如果出现 `确认 PIN 码` 页面，就按屏幕提示再输入一次相同 PIN。

## 修改开发板 PIN

路径：

```text
首页 -> 设置 -> PIN
```

操作步骤：

1. 进入 `设置`。
2. 选择 `PIN`。
3. 如果已经设置过 PIN，先看到 `输入 PIN 码`，这里输入当前开发板 PIN。
4. 进入 `PIN 设置`。
5. 选择 `修改 PIN`。
6. 看到 `设置 PIN 码` 后输入新的开发板 PIN。
7. 等待保存完成。

不要在这里输入智能卡 PIN。这里改的是开发板 PIN，不会改变 Satochip 或 SeedKeeper 卡片的 PIN。

## 开发板 PIN 输错会怎样

当前规则是连续输错 3 次触发保护。

| 连续输错次数 | 屏幕/设备行为 | 后果 |
|---|---|---|
| 第 1 次 | 显示 PIN 错误，并短暂等待 | 还有机会 |
| 第 2 次 | 显示 PIN 错误，并等待更久 | 还有最后一次机会 |
| 第 3 次 | 显示连续错误提示，清除本次会话，并尝试关机或重启 | 当前加载的钱包会从 RAM 中清掉，需要重新解锁或重新加载 |

重要说明：

| 问题 | 答案 |
|---|---|
| 第 3 次错会清空整机存储吗 | 当前构建不会把这当作“恢复出厂”来用；它会清除本次会话并关机/重启 |
| 正确输入后错误次数会怎样 | 正确输入会把开发板 PIN 错误次数清零 |
| 关机后错误次数会不会自动清零 | 不要依赖关机清零。看到错误次数接近 3 时，应停止尝试，确认 PIN 后再输入 |
| 忘记开发板 PIN 怎么办 | 不能找回。只能清理设备后用助记词或备份恢复钱包 |

## 自动锁定和自动关机

当前安全策略是固定的：

| 项目 | 当前值 | 行为 |
|---|---|---|
| 自动锁定 | 3 分钟无操作 | 清除本次敏感会话，回到开发板 PIN 解锁 |
| 自动关机保护 | 10 分钟无操作 | 清除本次敏感会话，并尝试关机；如果硬件不能软件关机，则重启 |

自动锁定后，输入的是开发板 PIN。

## 哪些地方要输入开发板 PIN

看到这些功能前后弹出 `输入 PIN 码`，输入开发板 PIN。

| 场景 | 输入哪个 PIN | 说明 |
|---|---|---|
| 设备开机 | 开发板 PIN | 解锁 Kern |
| 自动锁定后再次使用 | 开发板 PIN | 重新打开会话 |
| `首页 -> 设置 -> PIN` | 开发板 PIN | 先验证当前 PIN，才能修改设置 |
| 备份助记词、备份二维码、备份熵、钢板/点阵备份 | 开发板 PIN | 之后还会有敏感数据危险确认 |
| 查看本机扩展公钥 | 开发板 PIN | 保护本机钱包信息 |
| 查看本机地址 | 开发板 PIN | 保护本机钱包信息 |
| 管理本机助记词槽 | 开发板 PIN | 保护本机钱包信息 |
| BTC PSBT 签名 | 开发板 PIN | PIN 后还要再次核对交易并确认 |
| BTC 消息签名 | 开发板 PIN | PIN 后还要再次核对消息并确认 |

Web3 扫码签名时，如果你选择 `助记词` 作为签名来源，页面不会让你输入智能卡 PIN；如果页面没有出现 `输入 PIN 码`，就按确认页核对信息后继续。只要页面写着 `智能卡 PIN`，才输入智能卡 PIN。

## 智能卡 PIN 是什么

智能卡 PIN 是 Satochip 或 SeedKeeper 卡片自己的 PIN。它存在卡片里，Kern 开发板不知道也不能替你找回。

智能卡 PIN 用来保护：

| 功能 | 说明 |
|---|---|
| Satochip Web3 签名 | 卡片确认 PIN 后才会签名 |
| 生成 Web3 连接码 | 读取卡片账户信息时需要 PIN |
| 按路径查看 Satochip 地址 | 读取卡片路径地址时需要 PIN |
| 读取 BTC 观察公钥 | 读取卡片 xpub/ypub/zpub 等时需要 PIN |
| Satochip/SeedKeeper 维护 | 改卡 PIN、改标签、策略、重置等高风险操作需要卡 PIN |

开发板 PIN 和智能卡 PIN 互不影响：

| 事情 | 会影响谁 |
|---|---|
| 开发板 PIN 输错 | 只影响 Kern 开发板 |
| 智能卡 PIN 输错 | 只影响智能卡 |
| 修改开发板 PIN | 不会修改智能卡 PIN |
| 修改智能卡 PIN | 不会修改开发板 PIN |

## 什么时候输入智能卡 PIN

看到页面标题或输入框写着 `智能卡 PIN`，输入智能卡 PIN。

常见路径：

| 路径 | 输入哪个 PIN | 用途 |
|---|---|---|
| `连接钱包 -> Web3 -> 智能卡` | 智能卡 PIN | 生成 Satochip Web3 连接二维码 |
| `扫码签名 -> Web3 请求 -> 智能卡` | 智能卡 PIN | 用 Satochip 对 Web3 请求签名 |
| `智能卡工具 -> Satochip -> 地址` | 智能卡 PIN | 按路径读取地址 |
| `智能卡工具 -> Satochip -> BTC公钥` | 智能卡 PIN | 读取 xpub/ypub/zpub/tpub/upub/vpub |
| `智能卡工具 -> Satochip -> 维护` | 智能卡 PIN 或 PUK | 高风险维护，见下文 |
| `智能卡工具 -> SeedKeeper` 下需要卡授权的页面 | 智能卡 PIN | SeedKeeper 卡片操作 |

如果只是 `智能卡工具 -> Satochip -> 信息` 或读卡器检测，一般是读取状态，不要随便输入 PIN。先看页面有没有明确写 `智能卡 PIN`。

## 智能卡 PIN 输错会怎样

智能卡有自己的剩余次数。Kern 会读取卡片返回码，并显示类似：

```text
PIN 错误，剩余 2 次
```

或在状态页显示：

```text
PIN 剩余：PIN0=3 PUK0=5 PIN1=3 PUK1=5
```

具体剩余次数以你的卡片实际显示为准。

| 情况 | 后果 |
|---|---|
| 智能卡 PIN 输错但还有剩余次数 | 这张卡的 PIN 剩余次数减少 |
| 智能卡 PIN 剩余次数变成 0 | 卡片 PIN 锁定，Kern 会显示 `PIN 已锁定` 或类似错误 |
| PIN 锁定后继续签名 | 不会成功，必须先用 PUK 解锁 |
| 输错智能卡 PIN | 不会增加开发板 PIN 错误次数 |
| 输错开发板 PIN | 不会增加智能卡 PIN 错误次数 |

如果剩余次数只剩 1 次，不要继续猜。先确认卡片 PIN 记录，或者换到官方工具/维护流程处理。

## 修改智能卡 PIN

这是高风险操作。只有确认自己知道当前智能卡 PIN，并且知道卡片剩余次数足够时才操作。

路径：

```text
智能卡工具 -> Satochip -> 维护 -> 改PIN
```

操作步骤：

1. 插好读卡器和 Satochip 卡。
2. 进入 `智能卡工具`。
3. 进入 `Satochip`。
4. 进入 `维护`。
5. 选择 `改PIN`。
6. 在 `旧 PIN` 输入当前智能卡 PIN。
7. 在 `新 PIN` 输入新的智能卡 PIN。
8. `PIN 编号` 普通用户填 `0`。
9. 按 `执行`。
10. 结果显示 `PIN 已修改` 才算成功。

不要在 `旧 PIN` 输入开发板 PIN。这里改的是卡片 PIN。

SeedKeeper 卡片的改 PIN 入口类似：

```text
智能卡工具 -> SeedKeeper -> 改PIN
```

同样输入的是 SeedKeeper 卡片 PIN，不是开发板 PIN。

## 智能卡 PUK 和解锁

PUK 是智能卡 PIN 锁定后的救援码，不是普通 PIN。

路径：

```text
智能卡工具 -> Satochip -> 维护 -> 解锁
```

当前解锁流程用于 PIN0：

1. 进入 `解锁`。
2. 页面出现 `PUK`。
3. 输入智能卡 PUK。
4. 按 `执行`。
5. 结果显示 `PIN 已解锁` 才算成功。

风险说明：

| 情况 | 风险 |
|---|---|
| PUK 输错 | 智能卡 PUK 剩余次数减少 |
| PUK 剩余次数归零 | 可能无法正常解锁，需要官方工具、重置卡片或依赖助记词备份恢复 |
| 不知道 PUK | 不要猜，先停止 |

## Web3 签名时怎么选 PIN

扫码 Web3 请求后，先看签名来源。

| 你选择的来源 | 后续输入 |
|---|---|
| `助记词` | 不输入智能卡 PIN；核对确认页信息 |
| `智能卡` | 先核对确认页，再在 `智能卡 PIN` 页面输入智能卡 PIN |

签名前必须核对：

| 字段 | 要看什么 |
|---|---|
| 来源 | 是 `助记词` 还是 `智能卡` |
| 钱包/DApp | 是否是你正在操作的钱包和网页 |
| 链 | 是否是目标链 |
| 路径 | 是否是你要使用的账户路径 |
| 地址 | 是否是你要签名的地址 |
| 收款地址 | 是否正确 |
| 金额 | 是否正确 |
| Gas / Fee | 是否可接受 |
| 交易数据 | 如果看不懂或异常，不要签 |

TypedData/EIP-712 当前不会签名。页面会允许核对，但会提示暂未接入 EIP-712 哈希并停止签名。

## BTC 签名时怎么选 PIN

BTC PSBT 或 BTC 消息签名使用的是 Kern 本机助记词签名流程。

| 场景 | 输入哪个 PIN |
|---|---|
| PSBT 签名 | 开发板 PIN |
| BTC 消息签名 | 开发板 PIN |
| Satochip BTC 观察公钥 | 智能卡 PIN |

不要把 `Satochip BTC 观察公钥` 当成 `Satochip BTC 卡签名`。当前主线里 BTC 卡片签名不是同一个流程。

## 新手最容易错的地方

| 容易错的地方 | 正确做法 |
|---|---|
| 看到 PIN 就输入同一个 PIN | 先看标题：`输入 PIN 码` 是开发板，`智能卡 PIN` 是卡 |
| 用开发板 PIN 去签 Satochip | 错。Satochip 签名要智能卡 PIN |
| 用智能卡 PIN 去解锁设备 | 错。开机解锁要开发板 PIN |
| 卡 PIN 剩 1 次还继续猜 | 停止，不要把卡锁死 |
| PUK 当普通 PIN 使用 | 错。PUK 只用于解锁卡 PIN |
| 修改智能卡 PIN 时 `PIN 编号` 乱填 | 普通用户填 `0` |
| 签名确认页不看就点签名 | 必须核对地址、金额、链、路径和 DApp |

## 出问题时怎么判断

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 开机进不了主界面 | 开发板 PIN 错或还在等待倒计时 | 等倒计时结束，确认开发板 PIN 后再试 |
| 显示 `PIN 码连续错误 3 次` | 开发板 PIN 连续错 3 次 | 等设备关机/重启，再确认 PIN；不要继续猜 |
| Web3 智能卡签名失败，提示剩余次数 | 智能卡 PIN 错 | 停止猜，确认卡 PIN |
| 提示 `PIN 已锁定` | 智能卡 PIN 次数用完 | 需要 PUK 解锁 |
| 提示 `请输入智能卡 PIN` | 页面需要卡 PIN | 输入智能卡 PIN，不是开发板 PIN |
| 提示 `请先在设置里创建 PIN 码` | 开发板 PIN 还没设置 | 去 `设置 -> PIN` 或重启后按引导设置 |
| 不知道现在要输哪个 PIN | 页面标题不明确或自己不确定 | 返回上一页，重新从菜单路径判断 |

## 安全习惯

| 建议 | 原因 |
|---|---|
| 开发板 PIN 和智能卡 PIN 不要设成一样 | 一个泄露不应该影响另一个 |
| PIN 不要写在设备背面或卡套上 | 丢失设备/卡时会同时失守 |
| 卡 PIN 剩余次数低时不要继续尝试 | 锁卡后恢复成本高 |
| 保存好助记词备份 | 忘记 PIN、丢卡、坏卡时只能靠备份恢复 |
| 签名前逐项核对交易 | PIN 只能证明你授权，不能替你判断交易是否安全 |
| 不懂的维护项不要点 | 改标签、策略、重置、出厂等都可能影响卡片状态 |

## 一句话总结

看到 `输入 PIN 码`，输入开发板 PIN。看到 `智能卡 PIN`，输入智能卡 PIN。看到 `PUK`，只有在解锁智能卡时才输入 PUK。任何剩余次数接近 0 的情况都不要继续猜。
