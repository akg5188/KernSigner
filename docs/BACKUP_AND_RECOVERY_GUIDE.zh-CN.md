# 助记词备份与恢复指南

日期：2026-05-21

这份文档说明 KernSigner 里每种备份方式适合什么场景、怎么做、怎么验证、怎么恢复。新手只要记住一件事：创建钱包后先备份并演练恢复，再往里放任何资产。

当前固件仍是测试资金验收版。没有完成商业发布门禁、真机验收和独立审计前，不要导入真实助记词，不要存放真实资产；只用测试助记词、测试网资产或你明确可承受损失的小额验收资金。

## 备份前的安全规则

| 规则 | 原因 |
| --- | --- |
| 不拍照、不截图、不上传云盘 | 手机和云端最容易泄露 |
| 不把真实助记词输入联网网页 | 网页、插件、输入法都可能泄露 |
| 不只做一种备份 | 纸可能丢，金属可能刻错，文件可能损坏 |
| 备份后立刻恢复演练 | 没演练过的备份等于不确定 |
| 先小额测试 | 确认地址和签名链路正确 |

## 备份入口

常见路径：

```text
首页 -> 助记词 -> 当前助记词 -> 备份导出
```

如果页面要求 `输入 PIN 码`，输入开发板 PIN。这里不是智能卡 PIN。

## 备份方式怎么选

| 方式 | 优点 | 风险 | 推荐用途 |
| --- | --- | --- | --- |
| 明文助记词 | 最通用，任何钱包都能恢复 | 被看到就能被盗 | 必做基础备份 |
| 助记词二维码 | 恢复快，少打字 | 二维码等同助记词 | 只在安全环境临时使用 |
| 原始熵 | 可和 BIP39 工具精确验证 | 高级材料，泄露同样危险 | 验证和高级备份 |
| 钢板/1248/点阵 | 抗火水、适合长期保存 | 刻错不容易发现 | 长期冷备份 |
| Tinyseed/Stackbit | 适合对应实体板 | 需要按格式恢复 | 专用实体备份 |
| KEF 加密备份 | 可存储到卡或文件 | 密码丢失无法恢复 | 额外副本，不替代纸备份 |
| 智能卡写入/SeedKeeper | 卡内保存或生成 | 卡 PIN/PUK 管理复杂 | 熟悉智能卡后再用 |

## 明文助记词备份

操作：

1. 进入 `备份导出`。
2. 选择明文助记词或序号页面。
3. 用纸按顺序抄写全部单词。
4. 不要只抄单词，不抄序号。
5. 抄完后逐词核对一遍。
6. 把纸放到不会被拍照、不会被他人看到的位置。

检查：

1. 重新打开备份页面。
2. 用纸上的内容逐个核对。
3. 尤其检查相似词、漏词、顺序错误。

恢复演练：

1. 清除当前会话，或换一台测试设备。
2. 选择 `导入 -> 手动`。
3. 按纸上顺序输入。
4. 核对钱包指纹和地址是否一致。

## 助记词二维码备份

二维码包含助记词本身，风险和明文助记词一样高。

适合：

- 自己设备之间临时迁移。
- 安全离线环境下做恢复演练。

不适合：

- 打印后随便放。
- 用手机拍照保存。
- 发给别人或传网盘。

验证：

1. 用 KernSigner 的扫码导入功能扫回。
2. 或用离线工具解析。
3. 核对指纹和地址。

## 原始熵备份

原始熵是 BIP39 生成助记词前的核心随机数据。它泄露后等同于助记词泄露。

路径：

```text
首页 -> 助记词 -> 当前助记词 -> 原始熵
```

页面会显示：

| 字段 | 说明 |
| --- | --- |
| 指纹 | 当前钱包指纹 |
| 熵位数 | 128/160/192/224/256 位 |
| 原始输入 | 如果是骰子、硬币、扑克牌等方式创建，会显示输入材料 |
| 十六进制 | BIP39 原始熵 |

验证方法看 [MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md](MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md)。

建议：

- 可以抄十六进制作为高级备份。
- 抄写时每 4 或 8 个字符分组。
- 恢复时去掉空格即可。

## 金属备份

金属备份适合长期保存，但最怕刻错。

### 钢板/1248/点阵

操作前：

1. 确认使用的是测试助记词还是正式助记词。
2. 看清楚页面标题和格式。
3. 准备好实体板说明书。
4. 每刻一行就回读一遍。

恢复演练：

1. 用 `导入` 里的对应恢复方式。
2. 按实体板内容输入。
3. 恢复后核对指纹和前几个地址。

验收重点：

- 词数正确。
- 序号或点位没有偏一位。
- 恢复出来的指纹一致。
- 同一派生路径地址一致。

## KEF 加密备份

KEF 是加密备份。它适合做“额外副本”，不建议作为唯一备份。

操作建议：

1. 使用强密码。
2. 密码不要和开发板 PIN 相同。
3. 把 KEF 文件和密码分开放。
4. 备份后立刻做一次导入测试。

风险：

| 情况 | 后果 |
| --- | --- |
| KEF 文件丢失 | 无法从文件恢复 |
| 密码丢失 | 文件无法解密 |
| 文件和密码一起泄露 | 助记词可能泄露 |

## 智能卡备份和 SeedKeeper

智能卡相关功能涉及卡 PIN、PUK、剩余次数和导出权限。新手不要把它作为唯一备份。

建议流程：

1. 先用纸备份助记词。
2. 再测试智能卡读取状态。
3. 确认智能卡 PIN/PUK 记录正确。
4. 再考虑写入或从 SeedKeeper 加载。
5. 每次智能卡操作后记录卡片标签、SID、指纹。

智能卡 PIN 说明看 [SECURITY_PIN_GUIDE.zh-CN.md](SECURITY_PIN_GUIDE.zh-CN.md)。

## 恢复后必须核对什么

恢复成功后不要马上转账，先核对：

| 项目 | 怎么看 |
| --- | --- |
| 钱包指纹 | KernSigner 首页或助记词菜单 |
| 第 1 个收款地址 | KernSigner 地址页和观察钱包对比 |
| 扩展公钥 | KernSigner 公钥页和观察钱包对比 |
| 网络 | 主网/测试网必须一致 |
| 派生路径 | `m/84'/0'/0'`、`m/84'/1'/0'` 等必须一致 |
| BIP39 密语（passphrase，也叫密码短语） | 有无密语会得到完全不同钱包 |

如果指纹或地址不一致，先停下来，不要转入资金。

## 推荐备份组合

测试资金：

1. 明文纸备份。
2. BIP39 离线验证。
3. 一次恢复演练。

长期正式使用前的最低组合：

1. 纸备份一份。
2. 金属备份一份。
3. 可选 KEF 加密文件一份。
4. 两个不同地点保存。
5. 小额测试收款和签名。
6. 完成商业发布门禁后再考虑正式使用。

## 常见问题

### 只备份二维码可以吗

不建议。二维码损坏、格式不兼容或被拍照都很麻烦。纸质单词备份仍然最通用。

### 备份原始熵和备份助记词哪个更好

新手备份助记词。原始熵适合验证和高级备份。两者泄露风险一样高。

### 忘记密码短语怎么办

无法从助记词推回密码短语。用了密码短语就必须把它单独记好。

### 恢复出来地址不一样怎么办

按顺序检查：词是否一样、顺序是否一样、密码短语是否一样、网络是否一样、派生路径是否一样。

### 智能卡坏了怎么办

如果你有纸质助记词备份，可以恢复到新设备。如果只有智能卡，恢复取决于卡内导出权限、PIN/PUK 和卡片状态，所以不要只依赖智能卡。
