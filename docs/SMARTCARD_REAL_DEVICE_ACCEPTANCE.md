# 智能卡真机验收清单

日期：2026-05-22

目标：每次准备出固件前，用真机确认 Satochip 和 SeedKeeper 主线没有退化。本清单只验证当前测试资金验收构建已经开放的功能，不代表生产审计通过。

## 接线

- 开发板下载口接电脑。
- USB OTG 口接支持供电的转接线或带电 Hub。
- ACR39U-NF 插在 OTG 下游。
- Satochip 或 SeedKeeper 卡插入读卡器。
- 如果读卡器灯只闪一下，优先查外接供电，不先改软件。

## 基础检测

- 不插读卡器进入 `设备检查 -> 智能卡检测`，应显示无设备或超时错误。
- 插读卡器但不插卡，应识别 CCID，卡片上电应失败或提示无 ATR。
- 插卡后应拿到 ATR。
- 应能识别 Satochip 或 SeedKeeper AID。
- `读卡状态` 应能返回状态，不应蓝屏或重启。

## 连接钱包

- 进入 `连接钱包 -> OKX -> 智能卡账户`，输入 Satochip PIN，生成连接码。
- 手机 OKX 扫描连接码，能添加观察账户。
- 进入 `连接钱包 -> Bitget -> 智能卡账户`，输入 Satochip PIN，生成连接码。
- 手机 Bitget 扫描连接码，能添加观察账户。
- MetaMask、Rabby、TokenPocket 至少要能显示连接码；如果目标 App 不支持，文案不能误导。

## Web3 签名

- 进入 `扫码签名 -> 智能卡 -> 扫码 Web3`。
- 扫 OKX 测试资金转账请求。
- 设备显示钱包、类型、链、路径、地址。
- 输入 Satochip PIN 后生成签名结果二维码。
- 手机钱包扫描签名结果并能完成测试交易。
- 同一轮重进扫码，再签一次，确认相机不黑屏。
- Bitget 重复同样流程。
- personalSign 样本应成功或明确记录当前不支持。
- TypedData 必须拒签，不允许降级盲签。

## 路径和观察公钥

- `路径地址` 输入 `m/44'/60'/0'/0/0`，应显示 EVM 地址和二维码。
- BTC 地址路径必须输入完整地址路径，例如 `m/84'/0'/0'/0/0`。
- `观察公钥` 中主网 `zpub/ypub/xpub` 必须能显示和扫码。
- 测试网 `vpub/upub/tpub` 必须能显示和扫码。

## SeedKeeper 管理

- 空白或重置后的 SeedKeeper 进入 `设置PIN`，新 PIN 需要二次确认。
- 已设置 PIN 的 SeedKeeper 进入 `改PIN`，旧 PIN 和新 PIN 正确时应成功，新 PIN 需要二次确认。
- 从本机助记词写入 SeedKeeper 后，查看页应能列出卡内条目或钱包指纹。
- 从 SeedKeeper 查看页选择某个条目后，应能导入本机，并核对钱包指纹和首个地址。
- 保存密码后，列表里应能看到密码条目，并能查看或删除。
- 保存描述符后，列表里应能看到描述符条目，并能查看或删除。
- SeedKeeper 重置必须使用 `错PIN一步` 和 `错PUK一步`，直到返回 `FF00` 或提示空白卡。
- SeedKeeper 旧 `B0 FF` 流程返回 `9C20` 时，不能判定为读卡器坏，应提示使用新版错 PIN/错 PUK 流程。

## Satochip 维护

- 空白或出厂后的 Satochip 应能设置 PIN/setup。
- 已设置 PIN 的 Satochip 应能改 PIN，新 PIN 需要二次确认。
- 从本机助记词写入 Satochip 后，应能读取 EVM 地址或生成连接码。
- 重置种子或出厂重置前必须有确认提示，并要求用户知道这是清卡操作。
- 重置后应回到需要重新 setup 和写入助记词的状态。

## 异常路径

- 错 PIN：显示中文错误，不蓝屏。
- 拔卡后签名：显示读卡或 APDU 错误，不蓝屏。
- 断 Hub 电源：显示超时或读卡失败，不蓝屏。
- 签名中返回：不能留下 PIN 或卡响应缓存。
- 二维码解析失败：提示格式不支持，不重启。
- SeedKeeper 重置过程中输入正确 PIN 或正确 PUK 时，应提示重置中止，不能误清卡。

## 验收结果

只有以上主线全部通过，才允许说“智能卡测试资金主线可测”。即使全部通过，也不代表商业生产版完成。
