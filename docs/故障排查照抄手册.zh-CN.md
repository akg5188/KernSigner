# KernSigner 故障排查照抄手册

日期：2026-05-22

这份文档只做一件事：出问题时按表照抄排查。不要在不确定时反复输 PIN、反复全擦、反复刷错包。

## 先停三件事

遇到问题先停下来：

1. 不继续输入 PIN，特别是只剩最后一次机会时。
2. 不继续签名，特别是地址、金额、合约看不懂时。
3. 不直接全擦或乱刷，先保存日志和现象。

先记录这 6 项：

```text
1. 固件版本：
2. 硬件型号：
3. 当前页面标题：
4. 错误提示原文：
5. 刚才点了什么：
6. 是否有串口日志/照片/视频：
```

照片、截图、录屏不得包含助记词、原始熵、PIN、PUK、私钥或卡片恢复码；签名请求二维码也可能泄露地址、金额和 DApp 信息。排障材料用完后及时删除，并关闭自动云同步。

## 快速定位表

| 现象 | 先停什么 | 照做步骤 | 要收集的信息 |
| --- | --- | --- | --- |
| 开机黑屏 | 停止反复刷 app-only | 换线供电，确认 `wave_43`，抓串口日志，必要时刷 full 包 | 串口日志、刷机命令、固件文件名 |
| 花屏/白屏 | 不继续验收 | 确认板型和固件，调亮度，重启，抓日志 | 屏幕照片、板子型号 |
| 触摸不准 | 不继续输入助记词 | 重启，直接手指点，进 `自检 -> 外设 -> 触摸` | 触摸位置照片/视频 |
| 相机黑屏 | 不连续猛点 | 等 2-3 秒，返回重进，重启，跑相机自检 | 页面路径、日志 |
| 扫码扫不到 | 不判定签名失败 | 第一次扫小尺寸高密度签名码前，先用普通二维码调 OV5647 镜头焦距：10-20 cm，捏住镜头外圈轻轻扭动/旋转；再调亮度、避开反光、截图或投屏到电脑放大 | 二维码类型、钱包名、是否已对焦 |
| PSBT 签名失败 | 不广播 | 核对指纹、网络、找零、手续费，重新从第一帧扫 | PSBT 来源钱包、错误原文 |
| Web3 签名失败 | 不盲签 | 检查钱包、链、from、to、授权、二维码密度 | 钱包名、链、请求类型 |
| 连接钱包地址不一致 | 不收款 | 检查助记词、密语、路径、账户号、网络、来源 | KernSigner 地址、钱包地址 |
| 智能卡读不到 | 不猜 PIN | 检查带电 OTG、插口、读卡器灯、`自检 -> 智能卡` | 接线照片、ATR/错误 |
| SeedKeeper 重置返回 `9C20` | 不要当作供电问题 | 用错 PIN/错 PUK 到 `FF00` 的新版流程 | 返回码、卡类型 |
| PIN 不知道输哪个 | 不继续猜 | 看页面标题；`输入 PIN 码` 是开发板 PIN，`智能卡 PIN` 是卡 PIN | 页面照片 |
| 刷完还是旧版本 | 不连续乱刷 | 确认目录、串口、bin SHA256、刷入地址、是否重启 | `sha256sum`、刷机日志 |

## 1. 刷机问题

### 1.1 不知道刷 full 还是 app

按这个判断：

| 设备状态 | 应该刷 |
| --- | --- |
| 新板、空白板、未知固件 | full 包到 `0x0` |
| 已经运行 KernSigner，只升级应用 | app 包到 `0x20000` |
| 分区表不确定 | full 包 |
| bootloader 不确定 | full 包 |
| 只是 UI 或功能小更新，且设备已正常运行本项目 | app 包 |

不要把 app-only 包刷到空白板。

### 1.2 Linux 刷机照抄

```bash
cd /home/ak/123/KernSigner/firmware/wave_43
sha256sum -c SHA256SUMS.txt
./flash_wave43_linux.sh /dev/ttyACM0
```

如果 `/dev/ttyACM0` 不存在：

```bash
ls /dev/ttyACM*
ls /dev/ttyUSB*
```

### 1.3 刷机失败先查

1. USB 线是不是数据线。
2. 插的是下载口，不是 OTG 外设口。
3. 串口号是否正确。
4. 固件 SHA256 是否通过。
5. 是否被其他串口监视器占用。
6. 是否刷错 full/app。

## 2. 开机和屏幕问题

### 2.1 黑屏

照做：

1. 换 USB 线和电脑 USB 口。
2. 确认板子是 `Waveshare ESP32-P4-WiFi6-Touch-LCD-4.3`。
3. 确认刷的是 `wave_43` 固件。
4. 如果是新板，刷 full 包。
5. 抓串口日志。
6. 如果串口正常但屏幕黑，优先怀疑刷错板型、屏幕初始化或背光。

### 2.2 触摸不准

照做：

1. 重启设备。
2. 去掉厚保护膜或手套。
3. 进入：

```text
首页 -> 自检 -> 外设 -> 触摸
```

4. 记录哪些区域不准。
5. 确认不是刷了其他板型固件。

## 3. PIN 问题

### 3.1 判断输入哪个 PIN

| 屏幕文字 | 输入 |
| --- | --- |
| `输入 PIN 码` | 开发板 PIN |
| `设置 PIN 码` | 新开发板 PIN |
| `智能卡 PIN` | Satochip/SeedKeeper PIN |
| `旧 PIN` / `新 PIN`，在智能卡菜单里 | 智能卡旧/新 PIN |
| `PUK` | 智能卡 PUK |

### 3.2 开发板 PIN 忘了

做不到从设备里找回。你只能依赖助记词、KEF、智能卡或其他备份重新恢复。

先不要乱试。想清楚备份在哪里，再处理设备。

### 3.3 智能卡 PIN 剩一次

立刻停止：

1. 不再猜。
2. 找卡片 PIN/PUK 记录。
3. 只在确认后再输入。
4. 不确定就用官方工具或维护流程处理。

## 4. 助记词和地址问题

### 4.1 BIP39 网站结果不一致

按顺序查：

1. 网站语言是不是 `English`。
2. 是否离线打开。
3. 是否勾选 `Show entropy details`。
4. `Entropy Type` 是否正确。
5. `Mnemonic Length` 是否选 `Use Raw Entropy`。
6. KernSigner 上复制的是 `十六进制` 原始熵，不是别的字段。
7. 骰子、硬币、扑克牌是否漏输或多输。
8. D20 不要直接粘网站，先用 KernSigner 的十六进制原始熵验证。

### 4.2 恢复后地址不一样

按这个顺序排查：

| 检查 | 说明 |
| --- | --- |
| 单词 | 是否完全一样 |
| 顺序 | 是否顺序错 |
| 词数 | 12/15/18/21/24 是否一致 |
| 密语 | 有无 passphrase |
| 网络 | 主网/测试网 |
| 路径 | `m/84'/0'/0'` 等 |
| 地址类型 | legacy / nested SegWit / native SegWit / Taproot |
| 账户号 | account 0 还是 account 1 |

任何一项不一致，地址都可能不同。

## 5. 扫码问题

扫码问题的完整专项流程看 [扫码与摄像头排障手册](QR_CAMERA_TROUBLESHOOTING.zh-CN.md)。这里保留最短照抄版。

### 5.0 ESP32-P4 4.3 首次摄像头对焦

第一次使用这块开发板、换过摄像头、换过排线、装外壳或移动摄像头位置后，先做这一节。

照做：

1. 打开：

```text
首页 -> 自检 -> 外设 -> 相机
```

2. 用手机或电脑显示一个普通黑白二维码。
3. 距离保持 10-20 cm，屏幕亮度调高，避开反光。
4. 捏住 OV5647 摄像头镜头外圈，小幅扭动/旋转镜头。
5. 看到二维码边缘和小格子最清楚时停下。
6. 只扭镜头外圈；不要拧排线，不要拉摄像头主板，不要让 FFC 金手指松动。
7. 普通二维码稳定识别后，再测 OKX、Bitget、TokenPocket 这类高密度动态码。

判断：

| 现象 | 结论 |
| --- | --- |
| 普通二维码很快识别 | 焦距基本可用 |
| 普通二维码一直没反应 | 继续调焦距、距离、亮度、角度 |
| 只有 OKX 圆点码没反应 | 继续看 5.1 和扫码专项手册 |
| 有进度但不跳签名 | 图像层已识别，查分片、格式和签名解析 |

### 5.1 KernSigner 扫不到二维码

照做：

1. 先确认 5.0 的普通二维码对焦已经通过。
2. 手机亮度调高。
3. 开发板和手机保持 10-20 cm。
4. 二维码完整显示，不被边框挡住。
5. 不要斜着扫。
6. 避免强反光。
7. 如果二维码很小或很密，截图或投屏到电脑放大。
8. 如果是动态码，从第一帧开始扫完整。
9. 退回首页重新进入 `扫码签名`。
10. 还是失败就保存照片或录屏，先用电脑端 `zbarimg` / `components/zbar_qr/test` 验证。

### 5.2 手机扫不回 KernSigner 结果

照做：

1. 点 KernSigner 的大码显示或分片显示。
2. 调高 KernSigner 屏幕亮度。
3. 手机距离屏幕 10-20 cm。
4. 保证没有反光。
5. 确认手机钱包在“扫签名结果”页面，不是普通首页扫一扫。

## 6. BTC 问题

### 6.1 观察钱包导入后余额或地址不对

检查：

1. 导入的是 `zpub` 还是 `xpub`。
2. 手机钱包选择的地址类型是否一致。
3. 主网/测试网是否一致。
4. 是否用了密语。
5. 是否账户号不一致。

### 6.2 PSBT 签名失败

先看 KernSigner 提示：

| 提示/现象 | 处理 |
| --- | --- |
| 不是当前钱包 | 核对指纹和 xpub |
| 找零不认识 | 检查描述符、路径、钱包类型 |
| 网络不一致 | 主网和测试网不能混 |
| 二维码漏扫 | 重新从第一帧开始 |
| 手续费异常 | 取消，重新构造交易 |

不要为了通过流程而放宽检查。

## 7. Web3 问题

### 7.1 钱包连接扫不动

1. 确认手机钱包里选的是 `硬件钱包`、`Keystone`、`QR-based`。
2. 不要用普通首页扫一扫。
3. 不要选导入助记词。
4. KernSigner 上重新生成对应钱包二维码。
5. 确认来源是 `助记词` 还是 `智能卡`。

### 7.2 Web3 签名失败

按顺序查：

1. 钱包是不是 OKX、Bitget、MetaMask、Rabby、TokenPocket。
2. 请求类型是否当前开放。
3. TypedData/EIP-712 是否被拒绝；这是安全边界，不是普通故障。
4. 链 ID 是否正确。
5. from 地址是否是你的 KernSigner 账户。
6. 签名来源是否和连接来源一致。
7. 智能卡 PIN 是否正确且剩余次数足够。
8. 二维码是否太密，需要截图或投屏到电脑放大。
9. OKX 圆点动态码完全没反应时，先看 [QR_CAMERA_TROUBLESHOOTING.zh-CN.md](QR_CAMERA_TROUBLESHOOTING.zh-CN.md)，不要直接改签名协议；需要事故细节时再看 [OKX_QR_SCAN_INCIDENT_20260523.zh-CN.md](OKX_QR_SCAN_INCIDENT_20260523.zh-CN.md)。

## 8. 智能卡问题

### 8.1 读卡器灯只闪一下

优先判断供电不足。

照做：

1. 换带供电 OTG 转接线。
2. 或换真正外接供电的 USB Hub。
3. 确认读卡器在 Hub 下游。
4. 不要插下载口。
5. 再进入：

```text
首页 -> 自检 -> 智能卡
```

### 8.2 能看到 Hub，看不到读卡器

1. 确认 Hub 自己真的外接供电。
2. 换 Hub 下游 USB 口。
3. 换读卡器。
4. 换 OTG 方向。
5. 保存接线照片。

### 8.3 有 ATR，但 APDU 或签名失败

可能原因：

- 供电边缘不稳。
- 卡没插牢。
- PIN 错。
- 卡 applet 状态不对。
- 请求类型不是开放能力。

处理：

1. 重新插卡。
2. 重进 `自检 -> 智能卡`。
3. 先读状态，再做连接码。
4. 连接码成功后再测签名。
5. 失败时保存返回码。

### 8.4 SeedKeeper 重置

如果返回 `9C20`，不要当驱动坏。

新版 SeedKeeper 用：

```text
首页 -> 智能卡 -> SeedKeeper -> 重置
```

**重置会清空卡内所有秘密。只有确认已经有纸质或其他离线备份、并且这就是要清空的测试卡时才继续。**

然后：

1. 输入确定错误的 PIN，例如 `0000`。
2. 点 `错PIN一步` 到 PIN 锁定。
3. 输入确定错误的 PUK，例如 `0000`。
4. 点 `错PUK一步` 到返回 `FF00`。
5. 拔插卡。
6. 重新 `设置PIN`。

不要输入真实 PIN 或真实 PUK 来做重置。

## 9. 构建问题

### 9.1 固件构建失败

先确认：

```bash
cd /home/ak/123/KernSigner
git submodule update --init --recursive
. ~/esp/esp-idf-v5.5.4/export.sh
idf.py -B build_wave_43_fresh \
  -D SDKCONFIG=build_wave_43_fresh/sdkconfig \
  -D 'SDKCONFIG_DEFAULTS=sdkconfig.defaults;sdkconfig.defaults.wave_43' \
  build
```

常见原因：

- 没加载 ESP-IDF。
- ESP-IDF 版本不对。
- submodule 没拉。
- build 目录混用。
- 改了代码但刷的是旧 bin。
