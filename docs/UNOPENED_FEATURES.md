# 未开放功能红线

日期：2026-05-20

这些功能可能在 `/home/ak/123/satochip-signer` 参考项目里存在，但 KernSigner 当前不能作为已完成能力宣传，也不能放普通用户入口冒充功能。

## 智能卡

| 功能 | 当前状态 | 为什么不能开放 |
|---|---|---|
| Satochip BTC PSBT 签名 | 未迁移 | 和 Web3 摘要签名不是一套流程，必须处理 PSBT 输入、sighash、partial sig 和测试向量。 |
| Satochip BTC 消息签名 | 未迁移 | 需要独立 APDU、路径策略和签名格式验证。 |
| SeedKeeper 列表/导入/导出/删除 | 未迁移 | 涉及秘密材料和卡内对象生命周期，需要 PIN、擦除和失败恢复。 |
| 写助记词到 Satochip | 未迁移 | 不可逆高风险，错误会污染或覆盖卡内 seed。 |
| 写助记词到 SeedKeeper | 未迁移 | 涉及秘密材料持久化和导出策略。 |
| 修改 Satochip/SeedKeeper PIN | 未迁移 | 错误可能锁卡，必须有二次确认和恢复说明。 |
| 重置卡片 | 未迁移 | 极高风险，会清除卡内数据。 |
| 卡片初始化/import seed | 未迁移 | 会改变卡状态，必须单独设计。 |
| 卡片真伪证书检查 | 未迁移 | 未完成前不能宣传供应链认证。 |
| 2FA 管理 | 未迁移 | 2FA 卡当前必须拒绝，不能半支持。 |
| Satocash 管理 | 未迁移 | 参考项目还有独立 Satocash 分支，KernSigner 没有对应卡管理链。 |
| Satodime 管理 | 未迁移 | 参考项目还有独立 Satodime 分支，KernSigner 没有对应卡管理链。 |

## Web3

| 功能 | 当前状态 | 为什么不能开放 |
|---|---|---|
| TypedData/EIP-712 | 当前拒签 | 没有结构化哈希和可读字段显示，开放就是盲签风险。 |
| TokenPocket 原生交易码完整签名 | 部分支持 | 当前依赖 `request_sign_data_hex` 或 `eth-sign-request`，原生交易字段解析未完整对齐。 |
| 复杂合约方法解析 | 未完成 | 不能可靠显示合约方法、参数和授权风险。 |

## 商业生产

| 功能 | 当前状态 | 为什么不能开放 |
|---|---|---|
| 商业真钱包生产版 | 未达标 | `prodcheck` 未通过，Secure Boot、Flash Encryption、NVS 加密和调试面仍需生产配置。 |
| 真实资产验收 | 禁止 | 当前只允许测试助记词和测试资金。 |

## 开放原则

- 没有 C 端真实实现，不放入口。
- 没有真机验收，不放入口。
- 没有失败恢复，不放入口。
- 无法解析或无法让用户看懂的签名请求，拒签。
- 写卡、改 PIN、重置默认隐藏，必须二次确认。
