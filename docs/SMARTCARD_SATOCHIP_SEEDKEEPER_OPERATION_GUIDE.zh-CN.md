# Satochip / SeedKeeper 智能卡实测操作手册

日期：2026-05-22

这份文档记录本项目在 Waveshare ESP32-P4 4.3 寸开发板上反复真机测试后确认的智能卡流程。目标很实际：以后再刷机、换卡、换读卡器或给别人讲怎么用时，不再重新踩供电、PIN、重置和指纹这些坑。

当前固件仍按测试资金验收版使用。智能卡功能已经能跑通真实读卡和签名主线，但还不是经过第三方审计的商业生产钱包。

## 一句话结论

Satochip 是签名钱包卡，SeedKeeper 是秘密保存卡。当前所有智能卡流程都先用 PN5180 NFC 贴卡读取；如果 PN5180 硬件不可用，才使用 USB CCID 接触式读卡器作为备用。USB CCID 必须通过带外接供电的 OTG 转接线或外接供电 Hub，不能指望开发板 OTG 口直接给 ACR39U-NF 稳定供电。PN532 不再参与当前主固件运行路径。

## 必须这样接线

当前 NFC 主路线：

```text
PN5180 5V     -> 开发板 5V
PN5180 GND    -> 开发板 GND
PN5180 SCK    -> GPIO52
PN5180 MOSI   -> GPIO51
PN5180 MISO   -> GPIO50
PN5180 NSS/CS -> GPIO49
PN5180 BUSY   -> GPIO31
PN5180 RST    -> GPIO30
PN5180 IRQ    -> 不接
```

PN5180 详细接线、5V 注意事项和贴卡排错看 [PN5180 NFC 接线和使用说明](PN5180_NFC_WIRING_AND_USAGE.zh-CN.md)。

USB CCID 读卡器路线：

```text
ESP32-P4 USB OTG 口
-> 带供电 OTG 转接线 / 真正外接供电 Hub
-> ACR39U-NF Pocketmate II 读卡器
-> Satochip 或 SeedKeeper 智能卡
```

下载口只用于电脑串口和刷机，不是 USB Host 读卡口。

如果读卡器灯只闪一下、亮度很弱、键盘鼠标也不亮、Hub 能看到但读卡器看不到，优先判断为外设供电不足或 OTG 方向错误，不要先改驱动。

如果手机贴 PN5180 没有 NFC 感应，优先判断为 PN5180 供电、GND、模块焊接或天线摆位问题，不要先改 Satochip / SeedKeeper 上层逻辑。

## Satochip 和 SeedKeeper 区别

| 项目 | Satochip | SeedKeeper |
| --- | --- | --- |
| 主要用途 | 钱包签名卡 | 助记词、密码、描述符等秘密保存卡 |
| 常用功能 | Web3 连接、Web3 签名、路径地址、BTC 观察公钥、写入助记词 | 设置 PIN、改 PIN、保存助记词、查看/导入助记词、保存密码、保存描述符、重置 |
| 是否常插常用 | 签名时常用 | 备份/恢复时才常用 |
| 空白卡第一步 | 设置 PIN / setup | 设置 PIN / setup |
| 重置后状态 | 空白未 setup，需要重新设置 PIN 和写入助记词 | 空白未 setup，需要重新设置 PIN 和重新保存秘密 |

不要把 SeedKeeper 当成 Satochip 的另一个名字。它们是两个不同 applet，菜单也要分开理解。

## PIN 先分清楚

| 名称 | 保护谁 | 输错影响 |
| --- | --- | --- |
| 开发板 PIN | KernSigner 开发板和本机会话 | 连续错 3 次会清本次会话并关机/重启 |
| Satochip PIN | Satochip 卡 | 消耗 Satochip 卡自己的 PIN 次数 |
| SeedKeeper PIN | SeedKeeper 卡 | 消耗 SeedKeeper 卡自己的 PIN 次数 |
| PUK | 智能卡 PIN 救援码 | 输错会消耗 PUK 次数，归零后可能只能重置 |

看到 `输入 PIN 码`，通常是开发板 PIN。看到 `智能卡 PIN`、`旧 PIN`、`新 PIN`、`PUK`，就是智能卡自己的 PIN/PUK。

## 空白 Satochip 第一次使用

1. 先确认要走 PN5180 NFC 还是 USB CCID 读卡器。
2. 如果走 PN5180，把 Satochip 贴在 NFC 天线区域；如果走 USB，接好带供电 OTG、读卡器和 Satochip 卡。
3. 进入 `设备检查 -> 智能卡检测`，确认能识别到卡。
4. 进入 `智能卡 -> Satochip -> 设置PIN` 或维护里的 setup 入口。
5. 输入新 PIN，并按页面要求再次确认。
6. 记录好 PIN 和 PUK 信息，不要和开发板 PIN 混在一起。
7. 进入本机助记词页面，选择 `写入智能卡 -> Satochip`。
8. 写入完成后，用 `连接钱包` 或 `派生地址` 验证地址是否正确。

如果 Satochip 已经有种子，写入会被拒绝。需要先确认备份，再执行重置种子或出厂重置。

## 空白 SeedKeeper 第一次使用

1. 如果走 PN5180，把 SeedKeeper 贴在 NFC 天线区域；如果走 USB，接好带供电 OTG、读卡器和 SeedKeeper 卡。
2. 进入 `设备检查 -> 智能卡检测`，确认卡能被识别。
3. 进入 `智能卡 -> SeedKeeper -> 设置PIN`。
4. 输入新 PIN，并再次确认。
5. 进入本机助记词页面，选择 `写入智能卡 -> SeedKeeper`。
6. 写入完成后，进入 `智能卡 -> SeedKeeper -> 查看`。
7. 输入 SeedKeeper PIN 后，应看到卡内保存的条目或指纹。
8. 点选对应条目后，可查看、导入本机或删除。

SeedKeeper 平时可以只作为助记词保险箱使用，不需要每次签名都插它。

## 写入助记词到智能卡

推荐从已经确认过的本机助记词进入：

```text
助记词
-> 当前助记词
-> 写入智能卡
-> 选择 Satochip 或 SeedKeeper
```

写入前必须确认：

| 检查项 | 要确认什么 |
| --- | --- |
| 助记词 | 是当前要写入的那一组 |
| 密语 | 是否使用了 BIP39 passphrase |
| 指纹 | 与你记录的钱包指纹一致 |
| 卡类型 | Satochip 还是 SeedKeeper |
| 卡 PIN | 输入的是卡 PIN，不是开发板 PIN |

写入后不要只看“成功”两个字，必须再读取地址或查看卡内条目确认。

## 指纹以哪个为准

用来长期识别钱包的，应以“钱包指纹”为准。

钱包指纹通常是 BIP32 master fingerprint，也就是同一组助记词加同一个 BIP39 密语派生出来的钱包主指纹。它不是助记词文本的普通哈希，也不是 SeedKeeper 卡自己给 secret 生成的内部 ID。

判断规则：

| 显示项 | 是否可当钱包身份 |
| --- | --- |
| Kern 显示的钱包指纹 | 可以作为主要核对值 |
| 其他钱包显示的 master fingerprint | 同助记词、同密语时应一致 |
| BIP39 网站如果显示 BIP32 Root Fingerprint | 同助记词、同密语时应一致 |
| SeedKeeper 的 SID、secret id、内部 fingerprint | 可能只是卡内条目标识，不一定等于钱包指纹 |

本项目写入 SeedKeeper 时，应保存 Kern 计算出的钱包指纹用于人眼核对。不要把卡片内部 secret id 误认为助记词变了。

如果指纹不一致，先检查是否有密语、助记词顺序、单词序号、语言词库、派生工具口径不同。不要急着覆盖或重置卡。

## Satochip Web3 连接和签名

连接钱包：

```text
连接钱包
-> 选择 OKX / Bitget / MetaMask / Rabby / TokenPocket
-> 智能卡
-> 输入 Satochip PIN
-> 显示连接码
```

扫码签名：

```text
扫码签名
-> 自动打开相机
-> 扫手机钱包的签名请求
-> 选择智能卡
-> 核对交易信息
-> 输入 Satochip PIN
-> 显示签名二维码
```

已经真机跑通过的重点路径：

| 钱包 | 状态 |
| --- | --- |
| OKX | Web3 连接和测试转账签名已跑通 |
| Bitget | Web3 连接和测试转账签名已跑通 |
| MetaMask / Rabby / TokenPocket | 有入口，仍要继续收集更多二维码样本 |

如果手机钱包扫签名结果没有反应，优先看二维码格式、密度、亮度和距离，不要马上判断卡签名失败。

## SeedKeeper 查看和导入建议流程

SeedKeeper 查看页应该按这个流程使用：

1. 进入 `智能卡 -> SeedKeeper -> 查看`。
2. 输入 SeedKeeper PIN。
3. 页面列出卡内保存的条目，优先显示钱包指纹和标签。
4. 点选某个指纹或条目。
5. 再选择 `查看`、`导入本机`、`删除` 或其他操作。

导入本机后，要重新核对：

| 检查项 | 为什么 |
| --- | --- |
| 钱包指纹 | 确认导入的是同一组助记词 |
| 第一个地址 | 防止密语或路径口径不同 |
| 扩展公钥 | 防止账户类型选错 |

## SeedKeeper 保存密码和描述符

SeedKeeper 不只保存助记词，也可以保存密码、描述符等 secret。保存后应该能在卡内列表看到对应条目，并支持查看、修改或删除。

建议标签写短一点，例如：

```text
seed-main
okx-pass
btc-desc
```

不要把密码标签写成真实网站、真实账户名或敏感备注。

## SeedKeeper 重置正确流程

这是本次最重要的坑。

新版 SeedKeeper 不走旧的 `B0 FF` 拔插恢复出厂流程。旧流程会返回类似 `9C20`，不是 Kern 驱动坏了，而是这个命令在新版 SeedKeeper 上不适用。

新版 SeedKeeper 的恢复出厂方式是：故意让 PIN 锁定，再故意让 PUK 锁定。PUK 锁定到最后，卡会返回 `FF00`，然后恢复成空白卡。

菜单路径：

```text
智能卡
-> SeedKeeper
-> 重置
```

操作步骤：

1. 确认卡内助记词、密码、描述符都有别的备份。
2. 进入 `重置`。
3. `错 PIN` 输入一个确定错误的值，例如默认 `0000`。
4. 点 `错PIN一步`，重复到页面提示 PIN 已锁定或进入 PUK 阶段。
5. `错 PUK` 输入一个确定错误的值，例如默认 `0000`。
6. 点 `错PUK一步`，重复到返回 `FF00` 或提示已恢复为空白卡。
7. 拔出卡或断开读卡器，再重新插入。
8. 回到 `SeedKeeper -> 设置PIN`，重新 setup。

注意事项：

| 现象 | 含义 |
| --- | --- |
| `63Cx` | 输错，还剩 x 次 |
| `9C0C` | 当前 PIN 或 PUK 已锁，可以进入下一步 |
| `9C20` | 旧 B0FF 重置不支持，用错 PIN/错 PUK 流程 |
| `9C04` | 卡可能已经是未初始化空白状态 |
| `9000` | 你输入的值可能是正确 PIN/PUK，重置被中止 |
| `FF00` | 恢复为空白卡成功 |

重置时不要输入真实 PIN 或真实 PUK。要输入“确定错误”的值。

## Satochip 重置和出厂

Satochip 的重置种子和出厂重置与 SeedKeeper 不一样。Satochip Applet 仍有自己的重置/出厂流程，部分流程可能需要按屏幕提示反复拔插卡。

重置前先确认：

| 检查项 | 原因 |
| --- | --- |
| 助记词有纸质或其他备份 | 重置后卡内种子会消失 |
| 知道当前卡 PIN | 有些维护操作需要 PIN |
| 明白重置种子和出厂重置不同 | 重置种子清钱包种子，出厂重置会让卡回到更空白状态 |

重置后必须重新设置 PIN/setup，并重新写入助记词，才能继续连接钱包或签名。

## 常见报错

| 报错或现象 | 优先判断 | 处理 |
| --- | --- | --- |
| 读卡器灯只闪一下 | 外设供电不足 | 换带供电 OTG 或带电 Hub |
| 能看到 Hub，看不到读卡器 | 下游供电或方向不对 | 确认读卡器插在带电下游 |
| `9C20` | 命令不支持或流程错 | SeedKeeper 重置改用错 PIN/错 PUK |
| `63Cx` | PIN/PUK 错，剩余 x 次 | 不要继续猜，确认流程 |
| `FF00` | SeedKeeper 重置成功 | 拔插卡后设置 PIN |
| 写入后指纹看起来变了 | 可能看的是卡内 secret id | 以钱包指纹和地址交叉验证 |
| 签名一次后相机打不开 | 相机资源未释放或内存压力 | 退回首页重试，必要时重启并记录日志 |
| Web3 二维码过密扫不到 | ESP32 相机识别能力有限 | 截图或投屏到电脑放大；仍失败就保存照片/录屏做桌面端验证 |

## 每次出固件必须回归

1. 不插读卡器，智能卡检测不能蓝屏。
2. 插带供电读卡器不插卡，应有明确提示。
3. 插 Satochip，应能读取状态。
4. Satochip 设置/改 PIN 页面不能蓝屏。
5. Satochip 路径地址能选择助记词或智能卡来源。
6. Satochip OKX/Bitget Web3 连接码能被手机添加。
7. Satochip OKX/Bitget 测试转账签名能完成。
8. 插 SeedKeeper，应能读取状态。
9. SeedKeeper 设置 PIN、改 PIN、写入助记词、查看条目、导入本机要逐项测。
10. SeedKeeper 重置要用错 PIN/错 PUK流程，看到 `FF00` 才算成功。
11. 每个失败路径都只能显示错误，不能蓝屏重启。
12. 文档和 README 不能继续写已经过时的“SeedKeeper 只读、写卡未开放”。

## 不要再踩的坑

| 坑 | 正确做法 |
| --- | --- |
| 读卡器直插开发板 OTG | 用带供电 OTG |
| 把 `9C20` 当驱动坏 | 看是不是用了 SeedKeeper 旧重置命令 |
| 把卡内 secret id 当钱包指纹 | 用 Kern 钱包指纹和地址核对 |
| 设置 PIN 只输入一次 | 设置和改 PIN 都要二次确认 |
| 用开发板 PIN 当智能卡 PIN | 两者完全不同 |
| 菜单能点就当生产功能 | 必须真机验收和测试资金回归 |
