# 通用故障排查指南

日期：2026-05-21

这份文档用于快速定位常见问题：屏幕、触摸、相机、二维码、PIN、助记词、构建、刷机、连接钱包和智能卡。

## 先做三件事

1. 确认你使用的是测试助记词和测试资金。
2. 记录当前页面标题、你刚点了什么、错误提示原文。
3. 如果涉及刷机或崩溃，保存串口日志。

不要在不确定的情况下反复输入 PIN、反复刷机或全擦。

## 开机黑屏或花屏

先查：

| 检查项 | 怎么做 |
| --- | --- |
| 板型配置 | 当前项目只支持 `wave_43`，确认开发板是 ESP32-P4-WiFi6-Touch-LCD-4.3 |
| 供电 | 换数据线和 USB 口 |
| 串口日志 | 抓 boot log，看是否已经启动 |
| 背光 | 调亮度或换环境看屏幕 |
| 是否刷错包 | app-only 包不能刷空白板 |

如果串口正常但屏幕异常，优先怀疑刷错固件、不是 4.3 寸目标板，或显示初始化失败。

## 触摸不准或没反应

处理顺序：

1. 重启设备。
2. 用手指直接点，不隔着厚保护膜。
3. 检查是否刷错板型。
4. 进入设备检查或触摸测试页。
5. 保存日志和拍照记录。

如果只有某个页面无法返回，记录页面路径和截图。

## PIN 相关问题

先区分：

| 页面文字 | 输入什么 |
| --- | --- |
| `输入 PIN 码` | 开发板 PIN |
| `智能卡 PIN` | 智能卡 PIN |
| `PUK` | 智能卡 PUK |

连续输错开发板 PIN 3 次会清除本次会话并关机或重启。智能卡 PIN 输错会减少卡片自己的剩余次数。

详细说明看 [SECURITY_PIN_GUIDE.zh-CN.md](SECURITY_PIN_GUIDE.zh-CN.md)。

## 相机黑屏、卡住或返回不了

处理顺序：

1. 等 2-3 秒，不要连续猛点。
2. 点左上 `返回`。
3. 回到菜单后重新进入相机。
4. 如果还是黑屏，重启设备。
5. 保存串口日志。

相机创建助记词时，如果提示随机性不足，换一个细节更多的场景重新拍。

第一次使用 Waveshare ESP32-P4 4.3 开发板或刚换过摄像头/排线时，先做物理对焦：打开相机或扫码页，显示一个普通黑白二维码，保持 10-20 cm 距离，捏住 OV5647 镜头外圈小幅旋转，直到二维码边缘和小格子最清楚。不要拧排线、不要扯摄像头主板。普通二维码稳定识别后，再测 OKX、Bitget、TokenPocket 这类高密度动态码。

扫码失败时：

| 现象 | 可能原因 |
| --- | --- |
| 二维码太密 | 截图或投屏到电脑放大，或收集照片/录屏做桌面端解码 |
| 反光严重 | 调整屏幕角度和亮度 |
| 普通二维码都不灵敏 | 先调摄像头焦距、擦镜头、确认 FFC 排线插紧 |
| 一直扫不到 | 确认二维码类型是否支持 |
| 扫到但不能签 | 可能不是当前钱包、网络或路径不匹配 |
| OKX 圆点码完全没反应 | 先按 OKX 圆点动态二维码复盘检查摄像头焦距和图像解码层 |
| OKX 动态码有进度但不跳签名 | 查 UR 分片是否收齐、`eth-sign-request` CBOR 是否解析成功 |
| 刷完二维码好了但字体乱码 | 查字体是否为压缩位图，重新烘焙并确认 `.bitmap_format = 0` |

OKX 圆点动态二维码专项排障看 [OKX_QR_SCAN_INCIDENT_20260523.zh-CN.md](OKX_QR_SCAN_INCIDENT_20260523.zh-CN.md)。

## 助记词生成和 BIP39 网站不一致

先按这个查：

1. BIP39 网站语言是否是 `English`。
2. 是否勾选 `Show entropy details`。
3. `Mnemonic Length` 是否选了 `Use Raw Entropy`。
4. KernSigner 上复制的是 `十六进制` 原始熵，不是 SHA256 显示或别的输入。
5. 抛硬币、骰子、扑克牌是否漏输或多输。
6. D20 不要直接粘到 BIP39 网站，先用 KernSigner 的原始熵验证。

详细看 [MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md](MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md)。

## 恢复后地址不一样

按顺序排查：

| 检查项 | 说明 |
| --- | --- |
| 助记词 | 单词、顺序、词数必须一样 |
| 密码短语 | 有无 passphrase 会得到完全不同钱包 |
| 网络 | 主网和测试网地址不同 |
| 派生路径 | `m/84'/0'/0'` 与 `m/49'/0'/0'` 不同 |
| 地址类型 | legacy、nested SegWit、native SegWit、Taproot 不同 |
| 账户号 | account 0 和 account 1 不同 |

连接钱包教程看 [连接钱包教程.zh-CN.md](连接钱包教程.zh-CN.md)。

## BTC PSBT 签名失败

常见原因：

| 原因 | 处理 |
| --- | --- |
| PSBT 不是当前钱包 | 核对指纹和 xpub |
| 网络不匹配 | 主网/测试网必须一致 |
| 找零路径不认识 | 用受支持钱包和标准路径测试 |
| 二维码分片漏扫 | 重新从第一帧开始扫 |
| 手续费异常 | 取消，回钱包重新构造交易 |

签名前必须逐项核对收款地址、金额、找零和手续费。

## Web3 签名失败

先查：

1. 目标钱包是否是当前支持的钱包。
2. 请求类型是否已开放。
3. TypedData/EIP-712 是否被拒绝，这是预期安全边界。
4. 链 ID、from 地址、to 地址是否正确。
5. 签名来源是本机助记词还是智能卡。
6. 如果二维码太密，截图或投屏到电脑放大；仍失败就保存照片/录屏做桌面端解码。

不要为了“签过去”而忽略合约、授权和金额。

## 智能卡读不到

优先看 [TROUBLESHOOTING_SMARTCARD_POWER_OTG.md](TROUBLESHOOTING_SMARTCARD_POWER_OTG.md)。

快速检查：

| 检查项 | 说明 |
| --- | --- |
| 读卡器供电 | ACR39U-NF 推荐带电 OTG |
| 接口 | 不要把读卡器插到下载串口 |
| 卡方向 | 重新插卡 |
| PIN 剩余次数 | 不要连续猜 |
| 日志 | 运行 CCID probe 并保存 |

## 构建失败

先查：

1. ESP-IDF 是否已 source。
2. 是否缺 submodule。
3. 是否用错 ESP-IDF 版本。
4. 是否混用 build 目录。
5. 是否刚改了 C 文件但没跑语法检查。

常用命令：

```bash
cd /home/ak/123/KernSigner
git submodule update --init --recursive
source /home/ak/esp-idf-v5.5.4/export.sh
cmake --build build
```

更完整说明看 [BUILD_FLASH_DEBUG_GUIDE.zh-CN.md](BUILD_FLASH_DEBUG_GUIDE.zh-CN.md)。

## 刷机后还是旧版本

按顺序查：

1. 是否在正确目录构建。
2. 是否刷了刚生成的 `build_wave_43_fresh/kernsigner.bin`。
3. 是否刷到正确串口。
4. 是否只刷 app，但设备分区表不是同一套。
5. 是否刷完设备没有重启。
6. 是否实际启动的是旧 OTA 分区。

记录：

```bash
sha256sum build_wave_43_fresh/kernsigner.bin
```

把 SHA256 写入交付记录。

## 什么时候停止继续操作

遇到这些情况先停：

- 开发板 PIN 只剩最后一次机会。
- 智能卡 PIN 或 PUK 只剩最后一次机会。
- 收款地址和 KernSigner 显示不一致。
- 签名页面显示的金额、合约或授权看不懂。
- 相机或 QR 反复卡住并伴随崩溃日志。
- 不确定固件是不是测试版还是生产版。

停止后保存日志、截图、复现步骤，再继续排查。
