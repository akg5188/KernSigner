# KernSigner 功能菜单逐项索引

日期：2026-05-22

这份文档按设备屏幕菜单整理每个入口。它不是宣传清单，而是“看到按钮不知道做什么时”的索引。当前固件仍是测试资金验收版，智能卡维护、写卡、重置等入口属于高风险测试能力，不是已审计生产承诺。

## 扫码签名

常见路径：

```text
首页 -> 扫码签名
├─ OKX
│  ├─ 助记词
│  └─ 智能卡
├─ Bitget
│  ├─ 助记词
│  └─ 智能卡
├─ MetaMask
│  ├─ 助记词
│  └─ 智能卡
├─ Rabby
│  ├─ 助记词
│  └─ 智能卡
├─ TokenPocket
│  ├─ 助记词
│  └─ 智能卡
└─ BTC
   ├─ 助记词
   └─ 智能卡
```

支持用途：

| 扫到的内容 | 常见结果 | 注意 |
| --- | --- | --- |
| BTC PSBT | 核对交易后签名，显示签名结果二维码 | 优先用本机助记词主线 |
| Web3 请求 | 选择助记词或智能卡来源，核对后签名 | 看不懂合约/授权就取消 |
| 多片 UR/BBQR | 按页面提示继续扫完整 | 漏片会失败 |
| 消息签名请求 | 核对消息后签名 | 不签陌生登录消息 |
| 描述符/地址/公钥 | 解析或辅助导入 | 不等同签名 |
| 助记词二维码 | 可能进入导入或解析流程 | 二维码等同私钥材料 |

支持格式速查：

| 类别 | 支持内容 | 用途 |
| --- | --- | --- |
| 普通单帧 QR | 文本、地址、描述符、base64 PSBT、消息签名请求、助记词 | 基础扫码和解析 |
| `pNofM` 动态/分片 QR | `p1of3` 这类多帧文本 | 自动收齐后拼接 |
| UR | `ur:*` 单帧或多帧 | 处理 PSBT、描述符、bytes、Web3 请求等 |
| UR PSBT | `ur:crypto-psbt/...` | 解出 PSBT 后签名 |
| UR 描述符 | `ur:crypto-output/...`、`ur:crypto-account/...` | 提取或核对钱包描述符 |
| UR Web3 | `ur:eth-sign-request/...` | Web3 签名请求 |
| BBQR | `B$...` | 常用于高密度 PSBT 或交易数据 |
| TokenPocket 多片 | `tp:multiFragment-...` | 收齐并校验多片请求 |
| SeedQR | 4 位数字索引 | 助记词导入/备份 |
| Compact SeedQR | 16 或 32 字节二进制熵 | 12/24 词助记词导入/备份 |
| 明文助记词 QR | 空格分隔 BIP39 英文单词 | 临时导入或恢复演练 |

二维码格式能识别，不代表内容安全。签名前仍要在 KernSigner 上核对金额、地址、找零、手续费、链、合约和授权。

如果二维码太密，先确认 ESP32-P4 4.3 的 OV5647 摄像头已经用普通二维码调好焦，再截图或投屏到电脑放大；扫码和 OKX 圆点动态码排障看 [QR_CAMERA_TROUBLESHOOTING.zh-CN.md](QR_CAMERA_TROUBLESHOOTING.zh-CN.md)。

## 连接钱包

```text
首页 -> 连接钱包
├─ OKX
├─ Bitget
├─ MetaMask
├─ Rabby
├─ TokenPocket
├─ 派生地址
└─ BTC
```

### Web3 钱包

每个 Web3 钱包都有两种来源：

```text
首页 -> 连接钱包 -> OKX -> 助记词
首页 -> 连接钱包 -> OKX -> 智能卡
```

`OKX` 可以换成：

```text
Bitget / MetaMask / Rabby / TokenPocket
```

| 来源 | 含义 | 输入哪个 PIN |
| --- | --- | --- |
| `助记词` | 使用当前加载在 KernSigner 内存里的本机助记词 | 开发板 PIN |
| `智能卡` | 使用插在读卡器里的 Satochip 签名账户能力；SeedKeeper 主要保存秘密，导入本机后按 `助记词` 来源签名 | 智能卡 PIN |

手机钱包里要选 `硬件钱包`、`Keystone` 或 `QR-based`。不要导入助记词或私钥。

### 派生地址

```text
首页 -> 连接钱包 -> 派生地址
```

用途：

- 查看指定路径地址。
- 核对手机钱包地址是否和 KernSigner 一致。
- 排查网络、地址类型、账户号、密语差异。

### BTC

```text
首页 -> 连接钱包 -> BTC
├─ 助记词
│  ├─ zpub
│  └─ xpub
└─ 智能卡
   ├─ zpub
   └─ xpub
```

| 入口 | 用途 |
| --- | --- |
| `zpub` | 常用于 native SegWit 观察钱包 |
| `xpub` | 常用于 legacy 或部分兼容钱包 |
| `助记词` | 从当前本机助记词派生 |
| `智能卡` | 从智能卡读取观察公钥 |

BTC 收款前必须在 KernSigner 和观察钱包里核对同一个地址。

## 助记词

首页 -> 助记词
├─ 创建
├─ 导入
├─ 本机助记词
├─ 写卡
├─ 密语

### 创建

```text
首页 -> 助记词 -> 创建
├─ 骰子
├─ 抛硬币
├─ D20
├─ 扑克牌
├─ 手动
├─ 相机
├─ 十六进制
└─ 智能卡
```

| 方式 | 操作要点 |
| --- | --- |
| `骰子` | 输入六面骰 `1-6` |
| `抛硬币` | 输入 `0/1` |
| `D20` | 输入 `1-20` |
| `扑克牌` | 输入 `AH QS 9D TC` 这类牌面 |
| `手动` | 选择或输入 BIP39 单词 |
| `相机` | 对复杂画面取熵 |
| `十六进制` | 输入固定长度 hex 熵 |
| `智能卡` | 使用 SeedKeeper 等卡内随机能力 |

创建后先备份和恢复演练。

### 导入

```text
首页 -> 助记词 -> 导入
├─ 二维码
├─ 手动
├─ 序号
├─ 智能卡
├─ 存储卡
├─ 加密
├─ 钢板
└─ 点阵
```

| 方式 | 适合 |
| --- | --- |
| `二维码` | SeedQR / Compact SeedQR / 支持的助记词 QR |
| `手动` | 输入英文单词 |
| `序号` | 按 BIP39 编号恢复 |
| `智能卡` | 从 SeedKeeper 导入 |
| `存储卡` | 从 SD 或文件恢复 |
| `加密` | KEF 加密备份 |
| `钢板` | 金属备份恢复 |
| `点阵` | Tinyseed / Stackbit / 1248 类恢复 |

### 本机助记词

```text
首页 -> 助记词 -> 本机助记词
├─ 助记词
├─ 扫码签名
├─ 公钥
├─ 派生地址
├─ 备份
├─ 设置
└─ 清除会话
```

| 入口 | 用途 |
| --- | --- |
| `助记词` | 查看或管理当前加载的助记词流程 |
| `扫码签名` | 用当前助记词参与签名 |
| `公钥` | 查看扩展公钥或相关公钥 |
| `派生地址` | 按路径查看地址 |
| `备份` | 序号、原始熵、二维码、KEF、点阵、钢板、1248 |
| `设置` | 钱包相关设置 |
| `清除会话` | 从内存清除当前敏感会话 |

### 备份

```text
本机助记词 -> 备份
├─ 序号
├─ 原始熵
├─ 二维码
├─ 加密
├─ 点阵板
├─ 钢板
└─ 1248
```

所有备份入口都按敏感信息处理。不要拍照、不要截图、不要上传。

### 写卡

```text
首页 -> 助记词 -> 写卡
├─ SeedKeeper
└─ Satochip
```

写卡前确认：

- 当前助记词是要写入的那一组。
- 纸质备份已经做好。
- 智能卡 PIN/PUK 已记录。
- 读卡器接在带电 OTG/Hub 上。

### 密语

```text
首页 -> 助记词 -> 密语
```

用于设置 BIP39 passphrase。它会改变钱包指纹和所有地址。

### 高级

```text
首页 -> 助记词 -> 高级
├─ 助记词加密
├─ 派生地址
├─ BIP85
└─ 自检
```

高级功能适合已经理解助记词、派生路径、备份后果的人。新手先用测试助记词练习。

### Satochip

```text
首页 -> 智能卡 -> Satochip
├─ 信息
├─ 地址
├─ 写入
├─ BTC公钥
├─ 维护
├─ 高级
└─ 证书
```

| 入口 | 用途 | 风险 |
| --- | --- | --- |
| `信息` | 读取卡状态、版本、PIN 剩余次数 | 低 |
| `地址` | 按路径读取地址 | 中 |
| `写入` | 写当前助记词到卡 | 高 |
| `BTC公钥` | 导出 xpub/ypub/zpub/tpub/upub/vpub | 中 |
| `维护` | 设置 PIN、改 PIN、解锁、策略、重置 | 高 |
| `高级` | 认证钥、2FA、会话等后端能力 | 高 |
| `证书` | 导出/导入证书 | 高 |

Satochip `维护`：

```text
维护
├─ 设置PIN
├─ 改PIN
├─ 解锁
├─ 改标签
├─ NFC
├─ 功能
├─ 重置
├─ 出厂
└─ 真伪
```

这些入口涉及 PIN、PUK、策略和卡片数据。只在测试卡或明确维护场景使用。

Satochip `BTC公钥`：

```text
BTC公钥
├─ zpub
├─ ypub
├─ xpub
├─ vpub
├─ upub
└─ tpub
```

当前 BTC 卡片 PSBT 签名和 BTC 消息卡签名不要作为生产已开放能力宣传。

```text
首页 -> 智能卡 -> SeedKeeper
├─ 状态

├─ 改PIN
├─ 重置
├─ 剩余空间
├─ 列表
├─ 创建
├─ 写入
├─ 查看

├─ 存描述符

```

| 入口 | 用途 | 风险 |
| --- | --- | --- |
| `状态` | 看卡状态和条目信息 | 低 |
| `设置PIN` | 初始化空白卡 | 高 |
| `改PIN` | 修改卡 PIN | 高 |
| `重置` | 清空卡，恢复空白 | 极高 |
| `剩余空间` | 看可用空间 | 低 |
| `列表` | 列卡内条目 | 中 |
| `创建` | 用卡创建助记词 | 高 |
| `写入` | 写当前助记词到卡 | 高 |
| `查看` | 查看卡内助记词 | 极高 |
| `导入` | 从卡加载助记词到本机 | 高 |
| `存密码` | 保存密码条目 | 高 |
| `读描述符` | 读取钱包描述符 | 中 |
| `存描述符` | 保存钱包描述符 | 中 |
| `克隆` | 卡到卡条目克隆 | 极高 |
| `高级` | 主种子、2FA、派生密码、重置条目 | 极高 |

SeedKeeper `高级`：

```text
高级
├─ 主种子
├─ 2FA
├─ 派生密码
└─ 重置条目
```

普通新手不要把 SeedKeeper 作为唯一备份。先纸质备份，再测试卡功能。

## 当前能力边界

| 能力 | 当前口径 |
| --- | --- |
| BTC 助记词观察和 PSBT 流程 | 测试资金验收主线 |
| Web3 OKX/Bitget 连接和测试签名 | 已接测试主线 |
| MetaMask/Rabby/TokenPocket | 有入口，仍需更多样本回归 |
| Satochip Web3 连接/签名 | 测试卡验收范围 |
| SeedKeeper 写入/查看/导入/重置 | 测试卡验收范围 |
| TypedData/EIP-712 | 当前按拒绝/隐藏或非生产边界处理 |
| Satochip BTC PSBT 卡签名 | 不作为生产开放能力 |
| Satochip BTC 消息卡签名 | 不作为生产开放能力 |
| Secure Boot 生产锁定 | 文档是路线图/门禁，不是当前已完成证明 |

## 术语统一

| 文档里怎么写 | 含义 |
| --- | --- |
| 开发板 PIN | KernSigner 设备自己的 PIN |
| 智能卡 PIN | Satochip/SeedKeeper 卡自己的 PIN |
| PUK | 智能卡 PIN 救援码 |
| 助记词 | BIP39 单词，泄露等同私钥泄露 |
| 密语 | BIP39 passphrase，会改变钱包 |
| 观察钱包 | 只看地址和构造交易，不保存私钥 |
| 连接二维码 | KernSigner 给钱包扫，用来添加硬件/观察账户 |
| 签名请求二维码 | 手机钱包给 KernSigner 扫，用来请求签名 |
| 签名结果二维码 | KernSigner 给手机钱包扫，用来回传签名 |
