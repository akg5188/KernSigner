# 零基础第一次上手教程

日期：2026-05-21

这份教程给第一次接触 KernSigner 的人看。你不需要先懂硬件钱包、BIP39、PSBT 或智能卡，只要按步骤做，就能完成一次安全的测试流程。

当前 KernSigner 仍是测试资金验收版。没有完成商业发布门禁、真机验收和独立审计前，不要导入真实助记词，不要存放真实资产。

## 先认识四个角色

| 名称 | 它做什么 | 它不做什么 |
| --- | --- | --- |
| KernSigner 开发板 | 离线创建/导入助记词、显示地址、扫码签名 | 不联网广播交易 |
| 手机或电脑钱包 | 创建观察钱包、发起交易、广播交易 | 不应该保存 KernSigner 的助记词 |
| 电脑放大二维码 | 把太密或太小的 Web3 二维码放大给开发板扫描 | 不保存助记词、不替你签名 |
| BIP39 工具 | 离线验证助记词和原始熵是否一致 | 不要联网输入真实助记词 |

## 第 0 步：准备东西

你需要：

- KernSigner 开发板和 USB 数据线。
- 摄像头模组；第一次使用要先用普通二维码手动调焦。
- 纸和笔。
- 一台手机或电脑。
- 只用于测试的钱包或测试网络资产。
- 可选：智能卡和带供电的读卡器。

先把手机拍照、云同步、剪贴板自动上传这类习惯收一收。助记词一旦被拍到或上传，就等于泄露。

## 第 1 步：开机和设置 PIN

1. 插 USB 给 KernSigner 供电。
2. 等屏幕亮起。
3. 如果提示设置 PIN，设置一个你能记住但别人猜不到的开发板 PIN。
4. 如果提示输入 PIN，输入开发板 PIN。
5. 进入首页后，先不要连接真钱包。

注意：

- `输入 PIN 码` 是开发板 PIN。
- `智能卡 PIN` 是卡片自己的 PIN。
- 两者不是同一个东西。
- ESP32-P4 4.3 摄像头不是手机自动对焦。首次使用先打开相机或扫码页，普通二维码保持 10-20 cm，捏住 OV5647 镜头外圈小幅旋转到最清楚。

PIN 细节看 [SECURITY_PIN_GUIDE.zh-CN.md](SECURITY_PIN_GUIDE.zh-CN.md)。

## 第 2 步：创建一个测试助记词

第一次练习建议创建测试助记词，不要导入真实钱包。

路径：

```text
首页 -> 助记词 -> 创建
```

推荐新手先试两种方式：

| 方式 | 为什么推荐 |
| --- | --- |
| `16进制` | 最容易和 BIP39 工具逐字验证 |
| `相机` | 最接近日常使用，适合测试摄像头和随机熵流程 |

如果只是验证流程，可以用全 0 的 128 位十六进制熵：

```text
00000000000000000000000000000000
```

这会得到公开测试助记词，只能用于练习，不能用于真实资金。

更多创建方式和验证方法看 [MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md](MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md)。

## 第 3 步：用 BIP39 工具验证

验证的目标是确认：KernSigner 根据同一份原始熵生成的助记词，和 BIP39 工具生成的一样。

新手按这个做：

1. 下载并离线打开 BIP39 工具。
2. 语言选择 `English`。
3. 勾选 `Show entropy details`。
4. 在 KernSigner 上打开当前助记词的 `原始熵` 页面。
5. 把 KernSigner 显示的十六进制原始熵填进 BIP39 工具的 `Entropy`。
6. `Mnemonic Length` 选择 `Use Raw Entropy`。
7. 对比生成的英文助记词是否完全一致。

不要把真实助记词输入联网网页。详细截图式流程看 [MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md](MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md)。

## 第 4 步：先备份，再继续

创建助记词后，第一件事不是收款，而是备份和恢复演练。

最简单备份：

1. 进入当前助记词的备份页面。
2. 选择明文助记词。
3. 用纸按顺序抄写每个单词和序号。
4. 抄完后回读核对。
5. 清除当前会话，重新导入一次。
6. 核对钱包指纹和第一个地址是否一致。

不要只备份二维码，不要把助记词拍照，不要传网盘。

完整备份说明看 [BACKUP_AND_RECOVERY_GUIDE.zh-CN.md](BACKUP_AND_RECOVERY_GUIDE.zh-CN.md)。

## 第 5 步：连接观察钱包

观察钱包只能看地址和构造待签名交易，不保存私钥。

常见路线：

| 场景 | 怎么做 |
| --- | --- |
| BTC | KernSigner 显示 xpub/ypub/zpub，手机或电脑钱包导入观察账户 |
| Web3 | KernSigner 生成连接二维码，OKX/Bitget/MetaMask/Rabby/TokenPocket 扫码添加硬件账户 |
| 二维码太密 | 先确认摄像头已对焦，再截图或投屏到电脑放大，让开发板直接扫 |

收款前一定要在 KernSigner 上核对地址。手机里显示的地址和 KernSigner 上显示的不一致时，停止操作。

连接流程看 [连接钱包教程.zh-CN.md](连接钱包教程.zh-CN.md)。

## 第 6 步：做一次测试签名

BTC 测试：

1. 观察钱包创建一笔小额测试交易。
2. 钱包显示 PSBT 二维码。
3. KernSigner 扫码。
4. 在 KernSigner 上核对收款地址、金额、找零和手续费。
5. 输入开发板 PIN。
6. KernSigner 显示签名结果二维码。
7. 钱包扫回并广播测试交易。

Web3 测试：

1. 手机钱包用 KernSigner 账户发起测试交易。
2. 如果二维码太密，先确认摄像头已对焦，再截图或投屏到电脑放大，让 KernSigner 直接扫。
3. KernSigner 扫请求二维码。
4. 核对链、地址、金额、合约和授权。
5. 选择本机助记词或智能卡签名。
6. 手机钱包扫回结果。

看不懂交易内容时，不要签。

## 第 7 步：用完安全退出

1. 回首页。
2. 清除当前会话或关机。
3. 拔 USB。
4. 收好纸质备份。
5. 确认手机没有保存助记词照片。
