# 安卓构建环境准备

日期：2026-05-21

这份文档用于构建 `wallet/` 目录里的安卓中转/观察钱包 App。目标是让新手知道要装什么、怎么检查、怎么打 debug 包、怎么打 release 包。

这个 App 不是热钱包，不保存助记词或私钥。它主要用于观察账户、二维码中转、WalletConnect 协调和广播辅助。

| 项目 | 当前要求 |
| --- | --- |
| JDK | 17 |
| Android Gradle Plugin | 8.5.2 |
| Kotlin | 1.9.24 |
| Compile SDK | 34 |
| Target SDK | 34 |
| Build Tools | 35.0.0 |
| 最低 Android | API 26 |

## 推荐目录

```bash
cd /home/ak/123/Kern/wallet
```

如果你在另一个窗口构建，先确认目录：

```bash
pwd

```

看不到这些文件，就说明你不在 `wallet/` 目录。

## 安装 JDK 17

Ubuntu 常见安装方式：

```bash
sudo apt update
sudo apt install -y openjdk-17-jdk
```

检查：

```bash
java -version
javac -version
```

输出里应看到 `17`。

## 准备 Android SDK

如果你用 Android Studio：

1. 打开 Android Studio。
2. 进入 `SDK Manager`。
3. 安装 `Android SDK Platform 34`。
4. 安装 `Android SDK Build-Tools 35.0.0`。
5. 安装 `Android SDK Platform-Tools`。

如果你用命令行 SDK，把环境变量指向 SDK 目录：

```bash
export ANDROID_SDK_ROOT="$HOME/Android/Sdk"
export ANDROID_HOME="$ANDROID_SDK_ROOT"
export PATH="$ANDROID_SDK_ROOT/platform-tools:$PATH"
```

可选：在 `wallet/local.properties` 里固定 SDK 路径。

```bash
cd /home/ak/123/Kern/wallet
cp local.properties.example local.properties
```

然后把里面的 `sdk.dir=/path/to/Android/Sdk` 改成你机器上的真实路径。

## 安装 SDK 包

如果有 `sdkmanager`，可以执行：

```bash
sdkmanager "platforms;android-34" "build-tools;35.0.0" "platform-tools"
```

检查这些目录是否存在：

```bash
ls "$ANDROID_SDK_ROOT/platforms/android-34"
ls "$ANDROID_SDK_ROOT/build-tools/35.0.0"
ls "$ANDROID_SDK_ROOT/platform-tools"
```

## 构建 debug APK

新手和日常测试优先打 debug 包：

```bash
cd /home/ak/123/Kern/wallet
./scripts/build_local_ascii.sh debug
```

输出：

```text
dist/satochip-wallet-debug.apk
dist/satochip-wallet-debug.apk.sha256
```

如果从仓库根目录看，它们就是 `wallet/dist/satochip-wallet-debug.apk` 和对应的 `.sha256`。

也可以直接用 Gradle：

```bash
cd /home/ak/123/Kern/wallet
./gradlew :app:assembleDebug
```

直接 Gradle 的输出位置：

```text
wallet/app/build/outputs/apk/debug/app-debug.apk
```

## 构建 release APK

release 包必须配置正式签名，不能用 debug 签名冒充。

1. 准备一把长期保存的 Android keystore。
2. 复制配置模板：

```bash
cd /home/ak/123/Kern/wallet
cp keystore.properties.example keystore.properties
```

3. 编辑 `keystore.properties`：

```text
storeFile=/absolute/path/to/your-release.keystore
storePassword=你的store密码
keyAlias=release
keyPassword=你的key密码
```

4. 构建：

```bash
cd /home/ak/123/Kern/wallet
./scripts/build_local_ascii.sh release
```

输出：

```text
dist/satochip-wallet-release.apk
dist/satochip-wallet-release.apk.sha256
```

如果从仓库根目录看，它们就是 `wallet/dist/satochip-wallet-release.apk` 和对应的 `.sha256`。

如果没有 `keystore.properties` 或 keystore 文件不存在，release 构建会直接失败。这是安全设计。

## 校验 APK

```bash
cd /home/ak/123/Kern/wallet
sha256sum dist/satochip-wallet-debug.apk
cat dist/satochip-wallet-debug.apk.sha256
```

release 包把文件名换成：

```bash
sha256sum dist/satochip-wallet-release.apk
cat dist/satochip-wallet-release.apk.sha256
```

两边哈希必须一致。

## 安装到手机

先确认手机已打开开发者调试，并连接电脑：

```bash
adb devices
```

安装 debug 包：

```bash
adb install -r dist/satochip-wallet-debug.apk
```

如果手机安全策略不允许 debug、开发者选项、Root 或 test-keys，App 可能会主动拦截进入。这是当前高安全策略的一部分。

## 常见错误

| 现象 | 处理 |
| --- | --- |
| 找不到 Android SDK | 设置 `ANDROID_SDK_ROOT`，或填写 `wallet/local.properties` |
| 缺 `platforms/android-34` | 安装 `platforms;android-34` |
| 缺 `build-tools/35.0.0` | 安装 `build-tools;35.0.0` |
| Java 版本不对 | 安装并切换到 JDK 17 |
| release 构建失败 | 检查 `keystore.properties` 和 keystore 文件路径 |
| 构建很慢或内存不足 | 保持 `org.gradle.workers.max=1`，关闭其他大程序 |
| 改了代码但 APK 没变化 | 确认在 `/home/ak/123/Kern/wallet` 构建，重新安装刚生成的 APK |
