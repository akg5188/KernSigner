# KernSigner 全功能操作总手册

日期：2026-05-21

这份手册把 KernSigner 当前主要功能按“我想做什么”整理成入口地图。它不是商业生产承诺；当前固件仍按测试资金验收版使用。

如果你是第一次使用，先看 [小白照抄完整使用手册.zh-CN.md](小白照抄完整使用手册.zh-CN.md) 或 [零基础第一次上手教程.zh-CN.md](零基础第一次上手教程.zh-CN.md)，再回来看这份总手册。

如果你想按屏幕按钮逐项查功能，看 [功能菜单逐项索引.zh-CN.md](功能菜单逐项索引.zh-CN.md)。如果已经遇到问题，看 [故障排查照抄手册.zh-CN.md](故障排查照抄手册.zh-CN.md)。

## 总原则

- 先用测试助记词和测试资金。
- 先备份和恢复演练，再收款。
- 签名前必须在 KernSigner 上核对地址、金额、链、手续费、合约和授权。
- 看不懂就取消，不要为了流程通过而签名。
- 智能卡写卡、改 PIN、重置、SeedKeeper 管理已经进入测试卡验收范围，但不能当成已审计生产能力宣传；Satochip BTC PSBT 卡签名等仍未完成生产验收。

## 助记词创建

支持的创建方式包括：

| 方式 | 适合场景 |
| --- | --- |
| `十六进制` | 和 BIP39 工具精确验证 |
| `抛硬币` | 使用实体随机 0/1 |
| `骰子` | 使用六面骰 |
| `D20` | 使用二十面骰 |
| `扑克牌` | 使用洗牌后的牌面顺序 |
| `相机` | 使用相机画面提取随机熵 |
| `手动` | 输入已有 BIP39 单词 |
| `智能卡` | 使用卡片相关随机或账户能力 |

每种方式都要确认最终助记词通过 BIP39 校验。详细步骤看 [MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md](MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md)。

## 助记词导入

常见导入方式：

- 手动输入 BIP39 单词。
- 扫 SeedQR 或支持的助记词二维码。
- 输入 BIP39 单词序号。
- 从钢板、1248、点阵、Tinyseed、Stackbit 恢复。
- 从智能卡或 SeedKeeper 类流程加载。

导入后必须核对：

- 钱包指纹。
- 第一个收款地址。
- 扩展公钥。
- 网络和派生路径。
- 是否使用密码短语。

## 备份和恢复

推荐最低流程：

1. 明文纸备份。
2. BIP39 离线验证。
3. 清除会话后手动恢复一次。
4. 核对指纹和地址。
5. 再考虑金属备份、二维码备份、KEF 加密备份或智能卡备份。

完整说明看 [BACKUP_AND_RECOVERY_GUIDE.zh-CN.md](BACKUP_AND_RECOVERY_GUIDE.zh-CN.md)。

## 地址和扩展公钥

KernSigner 可以显示常见 BTC 地址类型、扩展公钥和自定义路径地址。

核对重点：

| 项目 | 常见差异 |
| --- | --- |
| 网络 | 主网和测试网不同 |
| 地址类型 | Legacy、Nested SegWit、Native SegWit、Taproot 不同 |
| 派生路径 | `m/84'/0'/0'` 和 `m/49'/0'/0'` 不同 |
| 密码短语 | 有无 passphrase 会得到完全不同的钱包 |
| 账户号 | account 0 和 account 1 不同 |

如果手机钱包地址和 KernSigner 地址不一致，不要收款。

## BTC 观察钱包和 PSBT

BTC 推荐路线：

1. KernSigner 显示 xpub/ypub/zpub。
2. 手机或电脑钱包导入观察账户。
3. 观察钱包生成收款地址。
4. KernSigner 核对地址。
5. 观察钱包生成 PSBT。
6. KernSigner 扫描、核对、签名。
7. 观察钱包扫回签名结果并广播。

PSBT 签名前必须确认：

- 输入属于当前钱包。
- 收款地址正确。
- 找零地址正确。
- 手续费合理。
- 网络一致。

## Web3 钱包连接和签名

当前连接钱包入口面向 OKX、Bitget、imToken、MetaMask、Rabby、TokenPocket 等 Web3 钱包场景。这些 Web3 钱包已用测试资金完成连接、扫码签名和签名结果回传验收；BTC 侧 BlueWallet 和 Electrum 的观察钱包/PSBT 签名流程也已实测通过。

常见路线：

1. KernSigner 生成连接二维码。
2. 手机钱包按硬件钱包或 Keystone 兼容流程扫码。
3. 手机钱包发起测试交易。
4. KernSigner 扫请求并核对摘要。
5. 使用本机助记词或 Satochip 智能卡签名；SeedKeeper 主要保存秘密，导入本机后按 `助记词` 来源签名。
6. 手机钱包扫回签名结果。

如果二维码太密，第一次签名前先确认 ESP32-P4 4.3 的 OV5647 摄像头已经用普通二维码调好焦：保持 10-20 cm，捏住镜头外圈轻轻扭动/旋转到最清楚。仍然难扫时，再截图或投屏到电脑放大，让 KernSigner 直接扫描。扫码和 OKX 圆点动态码排障看 [QR_CAMERA_TROUBLESHOOTING.zh-CN.md](QR_CAMERA_TROUBLESHOOTING.zh-CN.md)。

TypedData/EIP-712 等能力如果页面明确拒绝或隐藏，这是当前安全边界，不是普通故障。
