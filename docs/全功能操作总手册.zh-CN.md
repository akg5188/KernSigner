# Kern 全功能操作总手册

日期：2026-05-21

这份手册把 Kern 当前主要功能按“我想做什么”整理成入口地图。它不是商业生产承诺；当前固件仍按测试资金验收版使用。

如果你是第一次使用，先看 [零基础第一次上手教程.zh-CN.md](零基础第一次上手教程.zh-CN.md)，再回来看这份总手册。

## 总原则

- 先用测试助记词和测试资金。
- 先备份和恢复演练，再收款。
- 签名前必须在 Kern 上核对地址、金额、链、手续费、合约和授权。
- 看不懂就取消，不要为了流程通过而签名。
- 智能卡写卡、改 PIN、重置、SeedKeeper 管理、Satochip BTC PSBT 卡签名等未完成生产验收的能力，不能当成已开放功能宣传。

## 助记词创建

支持的创建方式包括：

| 方式 | 适合场景 |
| --- | --- |
| `16进制` | 和 BIP39 工具精确验证 |
| `抛硬币` | 使用实体随机 0/1 |
| `骰子` | 使用六面骰 |
| `D20` | 使用二十面骰 |
| `扑克牌` | 使用洗牌后的牌面顺序 |
| `相机` | 使用相机画面提取随机熵 |
| `手动` | 输入已有 BIP39 单词 |
| `智能卡` | 使用卡片相关随机或账户能力 |

每种方式都要确认最终助记词通过 BIP39 校验。详细步骤看 [MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md](MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md)。

## 助记词导入

常见导入方式：

- 手动输入 BIP39 单词。
- 扫 SeedQR 或支持的助记词二维码。
- 输入 BIP39 单词序号。
- 从钢板、1248、点阵、Tinyseed、Stackbit 恢复。
- 从智能卡或 SeedKeeper 类流程加载。

导入后必须核对：

- 钱包指纹。
- 第一个收款地址。
- 扩展公钥。
- 网络和派生路径。
- 是否使用密码短语。

## 备份和恢复

推荐最低流程：

1. 明文纸备份。
2. BIP39 离线验证。
3. 清除会话后手动恢复一次。
4. 核对指纹和地址。
5. 再考虑金属备份、二维码备份、KEF 加密备份或智能卡备份。

完整说明看 [BACKUP_AND_RECOVERY_GUIDE.zh-CN.md](BACKUP_AND_RECOVERY_GUIDE.zh-CN.md)。

## 地址和扩展公钥

Kern 可以显示常见 BTC 地址类型、扩展公钥和自定义路径地址。

核对重点：

| 项目 | 常见差异 |
| --- | --- |
| 网络 | 主网和测试网不同 |
| 地址类型 | Legacy、Nested SegWit、Native SegWit、Taproot 不同 |
| 派生路径 | `m/84'/0'/0'` 和 `m/49'/0'/0'` 不同 |
| 密码短语 | 有无 passphrase 会得到完全不同的钱包 |
| 账户号 | account 0 和 account 1 不同 |

如果手机钱包地址和 Kern 地址不一致，不要收款。

## BTC 观察钱包和 PSBT

BTC 推荐路线：

1. Kern 显示 xpub/ypub/zpub。
2. 手机或电脑钱包导入观察账户。
3. 观察钱包生成收款地址。
4. Kern 核对地址。
5. 观察钱包生成 PSBT。
6. Kern 扫描、核对、签名。
7. 观察钱包扫回签名结果并广播。

PSBT 签名前必须确认：

- 输入属于当前钱包。
- 收款地址正确。
- 找零地址正确。
- 手续费合理。
- 网络一致。

## Web3 钱包连接和签名

当前连接钱包入口面向 OKX、Bitget、MetaMask、Rabby、TokenPocket 等 Web3 钱包场景。

常见路线：

1. Kern 生成连接二维码。
2. 手机钱包按硬件钱包或 Keystone 兼容流程扫码。
3. 手机钱包发起测试交易。
4. Kern 扫请求并核对摘要。
5. 使用本机助记词或智能卡签名。
6. 手机钱包扫回签名结果。

如果二维码太密，使用 [ANDROID_RELAY_WALLET_GUIDE.zh-CN.md](ANDROID_RELAY_WALLET_GUIDE.zh-CN.md)。

TypedData/EIP-712 等能力如果页面明确拒绝或隐藏，这是当前安全边界，不是普通故障。

## 安卓中转 App

安卓 App 的定位是：

- 观察账户。
- 高密度 Web3 二维码中转。
- WalletConnect 协调。
- BTC xpub/ypub/zpub 观察账户原型。
- PSBT 请求准备和签名结果回传。

它不是热钱包，不保存助记词或私钥。

使用说明看 [ANDROID_RELAY_WALLET_GUIDE.zh-CN.md](ANDROID_RELAY_WALLET_GUIDE.zh-CN.md)。

构建环境看 [安卓构建环境准备.zh-CN.md](安卓构建环境准备.zh-CN.md)。

如果只想直接安装 APK，可以用本仓库 `wallet/dist/` 里的版本，也可以从 `satochip-signer` Releases 下载：

```text
https://github.com/akg5188/satochip-signer/releases
```
