# KernSigner 新手快速上手

日期：2026-05-21

这份文档给第一次拿到 KernSigner 开发板的人看。按这里走一遍，你会知道怎么开机、设置 PIN、创建或导入测试助记词、备份、收款、签名和安全退出。

当前固件仍是测试资金验收版。没有完成商业发布门禁、真机验收和独立审计前，不要导入真实助记词，不要存放真实资产；只用测试助记词、测试网资产或你明确可承受损失的小额验收资金。

## 你需要准备什么

| 物品 | 用途 |
| --- | --- |
| KernSigner 开发板 | 离线创建/导入助记词、显示地址、扫码签名 |
| USB 数据线 | 供电、刷机、抓串口日志 |
| 摄像头模组 | 扫二维码、相机随机创建助记词；首次使用要手动调焦 |
| 纸和笔 | 抄写助记词和备份信息；助记词与 PIN/PUK 分开记录、分开保存 |
| 手机钱包或电脑钱包 | 导入观察钱包、发起 PSBT 或 Web3 签名 |
| 智能卡和带电读卡器 | 只在需要 Satochip/SeedKeeper 时使用 |

## 第一次开机

1. 插 USB 供电。
2. 等屏幕进入 PIN 或首页。
3. 如果提示 `设置 PIN 码`，设置开发板 PIN。
4. 如果提示 `输入 PIN 码`，输入开发板 PIN。
5. 进入首页后，不要急着连接真钱包，先用测试流程熟悉操作。

开发板 PIN 和智能卡 PIN 不是一回事。看到 `智能卡 PIN` 时才输入卡 PIN。

第一次使用 ESP32-P4 4.3 摄像头时，先打开相机或扫码页，用普通黑白二维码在 10-20 cm 距离调焦：捏住 OV5647 镜头外圈小幅旋转，直到二维码边缘最清楚。普通二维码稳定识别后，再测 OKX、Bitget、TokenPocket 这类高密度动态码。

## 创建测试助记词

推荐先用 12 词测试助记词练习。

路径：

```text
首页 -> 助记词 -> 创建
```

可选方式：

| 方式 | 适合谁 | 说明 |
| --- | --- | --- |
| `十六进制` / `16进制` | 想和 BIP39 网站完全对照 | 最容易验证 |
| `抛硬币` | 想用实体随机 | 输入 0/1 |
| `骰子` | 有六面骰 | 输入 1-6 |
| `D20` | 有二十面骰 | 输入 1-20 |
| `扑克牌` | 有实体扑克牌 | 输入 `AH QS 9D TC` 这类牌面 |
| `相机` | 想快速生成随机熵 | 对复杂画面拍照 |
| `手动` | 已经知道要输入的词 | 要符合 BIP39 校验 |
| `智能卡` | 使用 SeedKeeper/Satochip 测试卡 | 随机数来自卡片；SeedKeeper 主要保存秘密，Satochip 主要签名 |

详细步骤看 [MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md](MNEMONIC_CREATION_BIP39_VERIFY.zh-CN.md)。

## 导入已有助记词

路径：

```text
首页 -> 助记词 -> 导入
```

常见导入方式：

| 方式 | 用途 |
| --- | --- |
| 手动输入 | 输入 12/15/18/21/24 个 BIP39 英文单词 |
| 扫码 | 扫 SeedQR 或支持的助记词二维码 |
| 数字/序号 | 输入 BIP39 单词编号 |
| 钢板/点阵/Tinyseed/Stackbit | 从实体备份恢复 |
| 智能卡 | 从 SeedKeeper/Satochip 加载 |

导入后先核对指纹、地址和扩展公钥。不要只看“导入成功”就开始转钱。

## 备份助记词

创建或导入成功后，先备份，再收款。

路径通常是：

```text
首页 -> 助记词 -> 当前助记词 -> 备份导出
```

常见备份方式：

| 备份 | 适合场景 |
| --- | --- |
| 明文助记词 | 最基础，必须抄清楚 |
| 助记词二维码 | 方便恢复，但更敏感 |
| 原始熵 | 用于 BIP39 验证和高级恢复 |
| 钢板/1248/点阵 | 长期实体备份 |
| KEF 加密备份 | 存储卡加密文件备份 |

详细步骤看 [BACKUP_AND_RECOVERY_GUIDE.zh-CN.md](BACKUP_AND_RECOVERY_GUIDE.zh-CN.md)。

## 收款前核对地址

收款不要只看手机钱包里显示的地址。要在 KernSigner 上核对。

推荐流程：

1. 手机或电脑钱包里创建观察钱包。
2. KernSigner 显示扩展公钥或连接二维码。
3. 手机/电脑导入观察钱包。
4. 手机/电脑生成收款地址。
5. KernSigner 上进入地址核对，确认地址一致。
6. 小额测试收款。

BTC 和 Web3 连接教程看 [连接钱包教程.zh-CN.md](连接钱包教程.zh-CN.md)。

## BTC 签名

常见流程：

1. 观察钱包创建交易。
2. 观察钱包显示 PSBT 二维码。
3. KernSigner 选择扫码签名。
4. KernSigner 扫 PSBT。
5. 核对收款地址、金额、找零、手续费。
6. 输入开发板 PIN。
7. 再次确认交易。
8. KernSigner 显示签名结果二维码。
9. 观察钱包扫回并广播。

只要收款地址、金额、手续费有任何不对，按返回取消。

## Web3 签名

常见流程：

1. 在 KernSigner 上进入连接钱包，选择 OKX、Bitget、MetaMask、Rabby 或 TokenPocket。
2. 选择签名来源：本机助记词或智能卡。
3. 手机钱包扫码添加账户。
4. 手机钱包发起测试交易。
5. KernSigner 扫请求二维码。
6. 核对链、地址、金额、合约、授权信息。
7. 选择签名来源；如果页面出现 `输入 PIN 码` 就输入开发板 PIN，如果页面写 `智能卡 PIN` 才输入智能卡 PIN。
8. 手机钱包扫回签名结果。

TypedData/EIP-712 等未开放能力不能当成已支持功能宣传。

## 使用密码短语

BIP39 密语（passphrase，也叫密码短语）会改变钱包指纹和地址。它不是“登录密码”，而是参与种子派生的额外要素；同一组助记词加不同密语会得到完全不同的钱包。

操作前先记住：

| 情况 | 结果 |
| --- | --- |
| 同一助记词 + 空密码短语 | 一个钱包 |
| 同一助记词 + `abc` | 另一个钱包 |
| 同一助记词 + `ABC` | 又是另一个钱包 |

如果你用了密码短语，备份时必须同时记住“助记词”和“密码短语”。丢任何一个都无法恢复同一个钱包。

## 安全退出

每次用完：

1. 回到首页。
2. 清除当前会话或关机。
3. 拔掉 USB。
4. 收好纸质备份。
5. 不要把助记词照片留在手机里。

设备自动锁定后也会清除敏感会话，但不要完全依赖自动锁定。

## 新手验收顺序

第一次拿到设备，建议按这个顺序：

1. 看 [SECURITY_PIN_GUIDE.zh-CN.md](SECURITY_PIN_GUIDE.zh-CN.md)，搞清楚开发板 PIN 和智能卡 PIN。
2. 用 `16进制` 创建全 0 测试助记词。
3. 用 BIP39 网站离线版验证助记词。
4. 查看原始熵、指纹、地址。
5. 做一次备份和恢复演练。
6. 创建观察钱包。
7. 做小额测试收款。
8. 做测试 PSBT 或 Web3 签名。
9. 再决定是否继续真机验收。
