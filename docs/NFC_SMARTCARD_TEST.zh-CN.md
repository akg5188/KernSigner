# NFC 智能卡签名测试说明

这版主固件加入了 PN532 NFC 智能卡通道。现有 USB CCID 读卡器路径保留，智能卡签名、读状态、改 PIN、写助记词等上层功能会走同一套 APDU 逻辑。

## 接线

先断电接线。接完不要马上插电脑上电，先做下面的安全检查：

```text
1. 只接 PN532 的 VCC/GND/SDA/SCL 四根线，IRQ/RSTO 先不接。
2. 确认 PN532 VCC 接开发板 3V3，不接 5V。
3. 确认 PN532 GND 接开发板 GND，不能接到旁边的信号脚。
4. 确认没有任何杜邦线跨到相邻针脚，排针没有歪插、错排、反插。
5. 万用表蜂鸣档检查 PN532 VCC 和 GND 不短路。
6. 第一次上电用手靠近模块，不要把模块装进外壳；一发热、冒烟、异味，马上拔 USB。
```

当前主固件默认使用 PN532 专用 I2C 总线，不再使用触摸屏共享的 `GPIO7/GPIO8`：

```text
PN532 VCC -> 开发板 3V3
PN532 GND -> 开发板 GND
PN532 SDA -> 开发板 GPIO28
PN532 SCL -> 开发板 GPIO31
```

实测结果：

```text
GPIO28/GPIO31: PN532 0x24 可扫到，固件版本可读取，智能卡 UID 可读取。
GPIO7/GPIO8: 与 GT911 触摸屏共用，之前智能卡 APDU 稳定性较差。
```

红色 PN532 V3 常见 I2C 拨码是 `SW1=ON`、`SW2=OFF`。改拨码后建议断电重新插 USB。

注意：不同卖家的 PN532 红板接口顺序可能不一样，不要只按图片或线的颜色接。必须看模块丝印上的 `VCC/GND/SDA/SCL`，再看开发板 40PIN 对应 GPIO 编号，逐个名字对名字接。即使模块页面写工作电压到 `5.5V`，也不要接开发板 `5V`，因为很多红板会把 I2C 上拉到供电电压，可能让 `SDA/SCL` 变成 5V 电平并伤到 ESP32-P4 GPIO。

## 如果模块发热或烧毁

立刻断电，不要继续反复插 USB。已经发热、冒烟、有异味、灯不亮或芯片鼓包的 PN532 不要再接开发板，避免把开发板 I2C 或 3V3 电源也拖坏。

优先检查这些原因：

```text
1. VCC/GND 插反。
2. VCC 接到了 5V 或错误电源脚。
3. 双排 40PIN 插错了一排，名字看对了但实际插在旁边那排。
4. 杜邦线公头歪了，短到相邻脚。
5. PN532 模块插到了 SPI 那排，或者模块两排接口丝印方向看反。
6. 便宜模块本身有虚焊、短路、稳压芯片质量差。
```

开发板自检顺序：

```text
1. 拔掉 PN532，只保留开发板 USB 下载口。
2. 看开发板能不能正常开机、屏幕能不能亮。
3. 电脑能不能识别串口。
4. 如果有万用表，量开发板 3V3 对 GND 是否约 3.3V。
5. 如果开发板能正常开机，再换新 PN532；不要用烧过的模块继续测试。
```

## 分阶段测试

新模块回来后按这个顺序测，不要第一步就进签名：

```text
第一阶段：只确认 PN532 没烧、I2C 在线
1. 只接 PN532，不贴智能卡。
2. 刷 nfc_bringup 小固件或主固件后看智能卡检测。
3. 目标是看到 I2C 0x24 和 PN532 firmware。

第二阶段：确认天线能读卡
1. 把智能卡贴近 PN532 白色天线区域。
2. 看 UID、SAK、ATQA。
3. Satochip/SeedKeeper 这类智能卡通常应显示 ISO-DEP/APDU: yes。

第三阶段：确认 Satochip/SeedKeeper APDU
1. 不插 USB 读卡器，只接 PN532。
2. 进入智能卡工具里的“智能卡检测”或“卡片信息”。
3. 目标是 SELECT applet、读取卡片状态或标签成功。
4. 主固件默认会等卡约 30 秒；卡已经贴好时会马上继续，不会硬等满。

第四阶段：再测签名和维护功能
1. 先测只读地址/公钥。
2. 再测小额测试签名。
3. 最后再测改 PIN、写助记词、重置这些维护功能。
```

正常时页面会显示：

```text
Selected: Auto
Active: NFC PN532
...
NFC PN532
Initialized: yes
PN532 present: yes
Card present: yes
ISO-DEP/APDU: yes
...
Card Label
...
```

如果 USB 读卡器和卡都已经插着，自动模式会优先用 USB。要专门测 NFC，先拔掉 USB 读卡器。

## 签名测试

NFC 读卡跑通后，原来的智能卡签名入口不用换流程：

- EVM 智能卡签名会通过同一套 Satochip 流程发送 APDU。
- BTC PSBT 智能卡签名也会复用同一套智能卡传输层。
- 改 PIN、设置 PIN、写入助记词、重置等维护功能同样走这个 transport。

注意：NFC 感应距离比 USB 读卡器敏感。签名过程中不要移动卡片，PN532 天线尽量贴近塑料外壳内侧，避开金属件、电池和大面积 PCB 铜皮。
